# SK Hynix 반도체 불량 예측 — 분석 결과 보고서

> **프로젝트**: FDC Trace 데이터 기반 Defect Test(C65) 회귀 예측  
> **평가 지표**: RMSE (Root Mean Squared Error, 낮을수록 좋음)  
> **모델**: LightGBM (Gradient Boosted Decision Trees)  
> **교차검증**: GroupKFold 5-Fold (웨이퍼 C64 기준 그룹 분리)

---

## 1. 데이터 개요

### 1.1 데이터 규모

| 구분 | 용도 | 설명 |
|------|------|------|
| **train_data.csv** | 모델 학습 | 정답(C65) 포함, 학습에만 사용 |
| **valid_X.csv** | 검증 (리더보드) | 모델 성능 중간 확인용 |
| **test_X.csv** | 최종 평가 | 최종 제출 및 순위 결정용 |

- 각 행은 **하나의 웨이퍼(C64)가 하나의 공정 단계(C7)를 거칠 때**의 센서 기록입니다.
- 하나의 웨이퍼는 여러 행을 가지며, Feature Engineering을 통해 **웨이퍼 단위(1 WF = 1 row)**로 집계됩니다.

### 1.2 컬럼 처리 요약

| 처리 | 컬럼 수 | 내용 |
|------|---------|------|
| 제거 | 22개 | Lot ID, 중복 ID, 상수, 전부 결측, 명세 제외, 원시 시간 |
| FDC 센서 (수치형) | 다수 | 통계 집계 대상 (mean, std, min, max, median, range, delta, slope) |
| 범주형 | 2개 | C6(레시피), C7(공정 단계) → 비율로 변환 |
| 메타 | 3개 | 행 수(n_rows), 총 시간(C41_total), PM 시간(C33) |

## 2. 모델링 방법론

### 2.1 Feature Engineering

원본 row-level 데이터를 웨이퍼 단위로 집계하여 다음 8가지 통계량을 추출했습니다:

| 통계량 | 설명 | 공정 관점의 의미 |
|--------|------|------------------|
| mean | 평균 | 전체 공정 동안의 센서 평균 수준 |
| std | 표준편차 | 센서 값의 안정성 (흔들림 정도) |
| min / max | 최솟값 / 최댓값 | 극단적 센서 상태 |
| median | 중앙값 | 이상치에 덜 민감한 대표값 |
| range | 범위 (max - min) | 센서의 전체 변동폭 |
| delta | 변화량 (끝 - 처음) | 공정 시작부터 끝까지의 순 변화 |
| slope | 기울기 (선형 회귀) | 시간에 따른 센서 추세 (상승/하강) |

추가 피처:
- **n_rows**: 웨이퍼당 공정 단계 수
- **C41_total**: 전체 공정 소요시간
- **C33_first / C33_max**: PM 후 경과시간 (장비 열화 지표)
- **C6 비율**: 레시피 구성 비율
- **C7 비율**: 공정 단계 구성 비율

### 2.2 모델: LightGBM

| 항목 | 설정 |
|------|------|
| 알고리즘 | GBDT (Gradient Boosted Decision Trees) |
| 학습률 (learning_rate) | 0.05 |
| 트리 복잡도 (num_leaves) | 63 |
| 최대 트리 수 (n_estimators) | 2,000 (early stopping 적용) |
| Early Stopping | 100 라운드 개선 없으면 중단 |
| 데이터 샘플링 (subsample) | 80% |
| 피처 샘플링 (colsample_bytree) | 80% |
| L1 정규화 (reg_alpha) | 0.1 |
| L2 정규화 (reg_lambda) | 1.0 |

### 2.3 교차검증: GroupKFold (5-Fold)

- **그룹 키**: 웨이퍼 ID (C64)
- **목적**: 같은 웨이퍼의 데이터가 학습과 검증에 동시에 포함되는 것을 방지 (data leakage 차단)
- **앙상블**: 5개 Fold 모델의 예측을 평균하여 최종 예측 생성

## 3. 성능 결과

### 3.1 RMSE 요약

| 평가 항목 | RMSE | 설명 |
|-----------|------|------|
| **베이스라인 (평균 예측)** | - | 모든 웨이퍼에 학습 데이터 평균값을 예측 (가장 단순) |
| **CV OOF (교차검증)** | - | 학습 데이터 내 교차검증 점수 |
| **Valid (검증셋)** | - | 리더보드 기준 점수 |
| **Test (최종 평가)** | - | 최종 제출 점수 |

> **참고**: 위 RMSE 수치는 노트북 실행 후 출력됩니다. 노트북의 **단계 6 (Valid / Test RMSE 평가)** 셀 출력을 확인하세요.

### 3.2 결과 해석 가이드

- **CV OOF RMSE와 Valid RMSE의 차이가 작을수록** 모델이 안정적입니다.
  - 차이 < 30: 안정적 (과적합 아님)
  - 차이 > 30: 과적합 의심
- **베이스라인 대비 RMSE가 크게 낮으면** 모델이 의미 있는 패턴을 학습한 것입니다.

### 3.3 Fold별 성능

| Fold | 역할 |
|------|------|
| Fold 1~5 | 각 RMSE와 최적 트리 수(best_iteration)가 출력됨 |
| 평균 / 표준편차 | Fold 간 RMSE의 평균과 변동폭 → 모델 안정성 확인 |

> Fold 간 RMSE 표준편차가 작을수록 모델이 데이터 분할에 덜 민감하여 안정적입니다.

## 4. Feature Importance (피처 중요도)

노트북 **단계 7**에서 마지막 Fold 모델의 Feature Importance 막대 그래프가 출력됩니다.

### 해석 방법

- 막대가 **긴(중요도가 높은) 피처**는 불량 예측에 가장 큰 영향을 미치는 센서입니다.
- 공정 엔지니어는 이 상위 피처에 해당하는 **센서를 우선적으로 모니터링**하면 불량을 줄이는 데 도움이 됩니다.
- 피처 이름의 접미사는 통계량을 나타냅니다:
  - `_mean`: 해당 센서의 평균
  - `_std`: 해당 센서의 변동성
  - `_slope`: 해당 센서의 추세
  - `_range`: 해당 센서의 변동폭
  - `_delta`: 해당 센서의 순 변화량

## 5. 일반화 점검

노트북 **단계 9**에서 3개의 그래프가 출력됩니다:

| 그래프 | 확인 내용 |
|--------|-----------|
| **Train: Actual vs OOF Pred** (히스토그램) | 학습 데이터에서 실제값과 예측값의 분포가 비슷한지 |
| **Valid: Actual vs Pred** (히스토그램) | 검증 데이터에서도 분포가 유사한지 |
| **Valid: Actual vs Predicted** (산점도) | 점들이 빨간 대각선(완벽한 예측) 근처에 모이는지 |

### 이상적인 결과

- 히스토그램: 실제값(파란색)과 예측값(주황색)의 모양이 비슷하게 겹침
- 산점도: 점들이 빨간 대각선 위에 밀집 → 예측이 정확함
- 산점도에서 점이 대각선에서 멀리 떨어진 부분 → 모델이 틀리는 영역

## 7. 모델링 설계 원칙

이 프로젝트에서 지킨 핵심 원칙과 그 이유입니다.

| 원칙 | 이유 |
|------|------|
| Lot ID(C20/C21/C22)를 피처에서 제외 | 새로운 Lot이 들어오면 모델이 알 수 없는 번호 → 일반화 불가 |
| WF ID(C64 등)를 피처에서 제외 | 웨이퍼 번호 자체는 물리적 의미 없는 식별자 |
| 물리적 의미가 있는 센서만 사용 | FDC 센서, 레시피, 스텝, PM 시간만이 실제 공정과 관련 있음 |
| train 데이터로만 학습 | valid/test 정답을 보고 학습하면 실전에서 성능 보장 불가 |
| GroupKFold(C64) 사용 | 같은 웨이퍼 데이터가 학습과 검증에 섞이면 성능이 부풀려짐 |

## 8. 한계 및 개선 방향

### 현재 한계

| 한계 | 설명 |
|------|------|
| 단일 모델 | LightGBM 하나만 사용. 다른 모델과 앙상블하면 성능 향상 가능 |
| 하이퍼파라미터 고정 | 현재 파라미터는 경험적으로 설정. 자동 탐색(Optuna 등) 미적용 |
| 피처 선택 미적용 | 중요도 낮은 피처 제거 시 과적합 감소 가능 |
| 시계열 순서 미반영 | Step 간 순서 정보를 더 정교하게 활용 가능 |

### 개선 가능 방향

1. **하이퍼파라미터 자동 탐색**: Optuna를 사용한 베이지안 최적화
2. **모델 앙상블**: XGBoost, CatBoost 등 다른 모델과 조합
3. **피처 선택**: 중요도 하위 피처 제거, Recursive Feature Elimination
4. **Target Encoding**: 레시피(C6)나 스텝(C7)에 대한 타깃 인코딩
5. **시계열 피처 확장**: 이동평균, rolling std, 구간별 통계 등

## 부록: 평가지표 RMSE 이해하기

RMSE(Root Mean Squared Error)는 "예측이 실제값에서 평균적으로 얼마나 벗어나는지"를 나타냅니다.

```
예시:
  실제값:  [100, 200, 300]
  예측값:  [110, 190, 310]
  오차:    [ 10, -10,  10]
  제곱:    [100, 100, 100]
  평균:    100
  RMSE:    sqrt(100) = 10.0

→ "평균적으로 10만큼 틀린다"
```

- RMSE = 0 이면 완벽한 예측
- RMSE가 작을수록 좋은 모델
- RMSE의 단위는 타깃(C65)의 단위와 동일하므로 직관적으로 해석 가능
