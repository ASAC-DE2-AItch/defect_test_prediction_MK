## 0. 한 줄 요약

**최종 운영 모델 = lean-85(85피처) + XGBoost(동결 파라미터) + 정기 재학습** — 2026-07-19 사용자 확정. 현업 기대 성능은 **미래 1주 pooled RMSE 99.84 · honest R² 0.8545**(주간 재학습 전제)이며, 요란 PM 직후 7일(레짐-온셋)은 어떤 모델도 못 맞추는 구간이라 `low_confidence` 플래그로 운영한다.

## 1. 확정 경위와 근거 (정직 기록)

경위: v13 M4에서 Conservative-GA(99) 셋 확정 → M5 XGB 튜닝 챔피언 → REPORT_12에서 시간불안정 14피처 제거한 **lean-85가 lot-CV 동급 + 파시모니로 채택 후보** → B2′ 롤링-재학습 최종 관문에서 lean-85(99.84)가 core10(98.36)에 1.48 열세 + R² 0.8545<0.9로 **G6′-B2′ 공식 기각** → v14 정상성 변환(P1~P4)까지 전부 무개선으로 종결. 통합 리포트의 순수 성능 권장은 core10이었다.

**그럼에도 lean-85로 확정한 근거** (사용자 결정, 2026-07-19):

1. **센서 실질 근거 요건(§0)**: 운영 모델에 FDC 센서가 실질적으로 포함되어야 한다는 팀 요건을 충족하는 구성 중 최선 — 필수 5센서(C17 온도·C11 플라즈마·C31 RF·C15/C16 가스) floor {2,5,3,3,1} 충족.
2. **same-era 신규 Lot에는 실질 이득**: lot-CV에서 core10 대비 −0.5 (66.956 vs 67.451), 챔피언99와 동급(66.956 vs 66.978) — 같은 시기 채점 축에서는 센서가 유효.
3. **파시모니·시간위생**: 챔피언99에서 드리프트 프록시 14피처(C25 9개 포함)를 제거해 lot-CV 손실 0으로 시간불안정 신호를 걷어낸 셋.

**트레이드오프 명시(정직)**: 시간축에서는 core10 대비 **+1.48 열세**(99.84 vs 98.36)가 실측이며, 이 격차는 재학습으로도 사라지지 않았다. 즉 lean-85 확정은 "성능 최적"이 아니라 "센서 포함 요건 하 최선"의 선택이다. 순수 시간축 성능이 우선순위가 되면 core10 전환을 재검토할 것.

## 4. 재학습 프로토콜 (운영 확정)

B2′ 동결 프로토콜을 운영으로 이식: **누적 데이터 전체로 재적합**(확장창), 주기는 **주간 7일 정기 + 요란 PM 기입 즉시 + PSI/RMSE 급등 수시**. 무재학습은 147→319 단조 발산으로 배포 불가가 실증됐으므로 재학습은 전제 조건이다. 구현: `lean85_pipeline.retrain()`(버전 저장+manifest) · `should_retrain()`(트리거) · `walkforward_acceptance()`(B2′ 재현 수용검사, 기준 **pooled 99.840 ± 0.5**, 위 fold표가 기준선). 온셋 7일은 `low_confidence=1`로 플래그.

## 5. 수치 인용 규칙

현업/보고 = **시간축 99.84 / R² 0.8545**만. lot-CV 66.8~67.0은 same-era(후향·대회축) 전용 — 미래를 엿본 낙관치로 현업 인용 금지. 254.9는 재학습 필수의 근거로만.

## 6. 한계·리스크

1. **정확도 바**: 재학습해도 미래 1주 R² ≈ 0.85 — 통합 리포트의 목표(70) 대비 약 30pt 부족은 현 데이터·현 센서로 해소 불가 판정(v14 종결). lean-85도 이 천장 안에 있다.
2. **레짐-온셋 예측 불가**: 요란 PM 직후 주는 구조적으로 못 맞춘다(147/113/109) — 플래그 운영이 유일한 완화책.
3. **시간축 열세 1.48**: core10 대비 열세는 센서 유지의 비용. 성능 우선 전환 옵션은 §1 트레이드오프 참조.
4. **검증 범위**: 시간 견고성 근거는 학습기 내부 rolling — 진짜 미래(새 캠페인) 데이터로 초기 수 주간 병행 검증 권장.
5. **신규 레시피**: C6_2 등 등장 시 is_special_recipe=0 보수 폴백 — 특수형이면 매핑 갱신 필요. 신규 요란 PM은 pm_log 기입이 전제(미기입 시 days 피처 왜곡).
