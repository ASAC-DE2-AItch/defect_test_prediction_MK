# 실무용 모델, 어떤 방향으로 갈 것인가 — PPT 보조 자료 (3장 이내)

> **정직 모델 기준**(Lot 누수 배제 = 신규 Lot에도 일반화되는 실제 성능). 출처: `산포도_비교_최종_보고서.md`
> 슬라이드는 팀원이 제작. 이 문서는 **넣을 내용(제목·이미지·문구·수치)** 만 정리한 핸드오프입니다.

**한 줄 결론:** 시간피처 only는 위험, 센서피처 only는 부정확 → **시간+센서 결합(combined)을 실무 base로 채택**, 단 피처가 너무 많으니 **피처 다이어트로 정확도를 끌어올린다.**

## 슬라이드 1 · 3안 비교 & 결정
- **이미지:** `1_direction.png`
- **핵심 문구**
  - 시간피처 중심(코어10·10피처): test 39.04 (정직 최저)이지만 **10개 중 7개가 시간·PM달력 피처** → 센서가 실질 근거가 못 됨, 공정변화 취약·원인진단 불가 = **실무 위험**
  - 센서피처 중심(modeling_v5·185피처): test 60.52 → 센서 집계만으론 레짐 내부 변동을 못 잡음 = **정확도 부족**
  - 시간+센서 결합(combined FULL·192피처): test 49.15 → 두 축 모두 반영해 균형 = **채택 방향** (단 피처 과다)

## 슬라이드 2 · 근거 (성능 비교)
- **이미지:** `2_comparison.png`
- **핵심 문구**
  - 정직 test RMSE: **코어10 39.04 < combined 49.15 < v5 60.52**
  - 코어10이 낮지만 시간 편중이라 실무 위험 / v5는 센서 기반이나 부정확 / combined는 균형이라 **다이어트로 개선할 대상**
  - ※ v9·v10 등 Lot 누수 라인(32~34)은 신규 Lot에서 70~100으로 붕괴 → **실무 제외**(별도 보고 완료)
- **(선택) 산포도 증빙 이미지** — 상위 폴더 `산포도 개형/` 에 있음:
  - 코어10: `PM 이후 레짐·시간 파생 피 산포도 개형.png`
  - v5: `modeling_v5 산포도 개형.png`
  - combined: `combine피쳐 산포도 개형.png` (또는 `image.png`)

## 슬라이드 3 · 실행 계획 (피처 다이어트)
- **이미지:** `3_diet_roadmap.png`
- **핵심 문구**
  - 현재 combined FULL 192(valid 48.76 / test 49.15) = 시간·레짐 7 + 센서 집계 180여
  - **유지:** 시간·레짐 7 + 핵심 센서(C17 온도, C59/60, C61/62, C12, C11)
  - **제거:** 저기여·중복 WF 집계 180여(대부분 노이즈)
  - **목표:** 정확도↑(코어10급 ≈39 근접·초과) + 센서가 실질 근거(실무 진단 가능·신규 Lot 견고)
  - 근거: combined 192는 시간 신호는 충분한데 센서 집계 다수가 노이즈 → 코어10보다 9pt 뒤짐. **"많은 피처 ≠ 좋은 피처"**
