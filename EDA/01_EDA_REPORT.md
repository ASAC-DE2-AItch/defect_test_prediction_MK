## 3. 웨이퍼(WF) 그룹 구조

### 웨이퍼당 행 수 통계

| 항목 | 값 |
|------|---|
| 최소 | 9행 |
| 최대 | 24행 |
| 중앙값 | 10행 |
| 표준편차 | 1.12 |

- 대부분의 웨이퍼가 **약 10개의 공정 단계(행)** 를 가집니다.

### C65 상수 여부

- **C65는 같은 웨이퍼 내에서 항상 동일한 값입니다: True**

### 조치 방향

> 웨이퍼 1개 = 1행으로 만든 뒤 모델에 입력하는 전략이 적합합니다.

## 4. 예측 대상(C65) 분포

### 기초 통계

| 통계량 | Train | Valid | Test |
|--------|-------|-------|------|
| 개수 | 11,939 | 1,990 | 1,990 |
| 평균 | 973.21 | 973.26 | 977.89 |
| 표준편차 | 261.71 | 259.04 | 261.98 |
| 최솟값 | 503.00 | 511.00 | 503.00 |
| 25% | 670.50 | 673.00 | 672.00 |
| 50% (중앙값) | 1,012.00 | 1,014.00 | 1,014.00 |
| 75% | 1,138.00 | 1,131.00 | 1,140.00 |
| 최댓값 | 1,678.00 | 1,672.00 | 1,643.00 |
| 왜도(Skewness) | 0.05 | 0.07 | 0.03 |
| 첨도(Kurtosis) | -0.80 | -0.68 | -0.82 |

### 해석

- **평균 약 973**: 세 데이터셋 모두 거의 동일한 평균값을 보입니다.
- **왜도 ≈ 0**: 분포가 좌우 대칭에 가깝습니다. 한쪽으로 치우치지 않았습니다.
- **첨도 ≈ -0.8**: 정규분포보다 약간 납작한 형태입니다 (양쪽 꼬리가 가벼움).
- **범위 503 ~ 1,678**: 결함 수치가 약 500에서 1,700 사이에 분포합니다.

### 조치 방향

> 왜도가 0에 가까우므로 **로그 변환 등 타깃 변환은 불필요**합니다.
> 분포가 비교적 균등하여 그대로 사용해도 됩니다.

## 5. 시간에 따른 변화 (Temporal Drift)

### 결과

| 구분 | 시작 시점 | 종료 시점 |
|------|----------|----------|
| Train | 2018-12-01 00:44 | 2019-02-08 09:43 |
| Valid | 2018-12-01 00:46 | 2019-02-08 09:41 |
| Test | 2018-12-01 00:57 | 2019-02-08 09:35 |

### 해석

- 세 데이터셋의 시간 범위가 **거의 완전히 겹칩니다**.
- 시간 순서대로 분할한 것이 아니라, **랜덤 분할**로 나눈 것으로 확인됩니다.
- 이는 시간에 따른 분포 변화(temporal drift)가 모델에 큰 영향을 주지 않을 것임을 의미합니다.

### 조치 방향

> 시간 변수를 특별히 처리할 필요는 없으나, 시간 기반 파생 변수(예: PM 이후 경과시간)는
> 여전히 유용할 수 있습니다.

## 6. 데이터 분포 변화 (Covariate Shift)

### Adversarial Validation AUC 결과

| 비교 쌍 | AUC | 해석 |
|---------|-----|------|
| Train vs Valid | **0.5103** | 거의 구분 불가 = 분포 동일 |
| Train vs Test | **0.5008** | 거의 구분 불가 = 분포 동일 |
| Valid vs Test | **0.4970** | 거의 구분 불가 = 분포 동일 |

> AUC가 0.5면 동전 던지기 수준이므로 "구분할 수 없다" = "분포가 같다"는 뜻입니다.

### 해석

- 세 데이터셋 모두 AUC가 **0.50 부근**으로, 센서값의 분포가 거의 동일합니다.
- 이것은 매우 좋은 신호입니다. Train에서 학습한 패턴이 Valid/Test에서도 잘 작동할 가능성이 높습니다.

### 조치 방향

> 도메인 적응(domain adaptation) 같은 특수한 기법은 **불필요**합니다.
> 일반적인 머신러닝 파이프라인을 그대로 적용하면 됩니다.

## 7. 센서값과 결함의 상관관계

### 피어슨 상관계수 |r| > 0.3 인 피처 (4개)

| 피처 | 피어슨 상관계수 | 관계 방향 |
|------|--------------|----------|
| **C17** | **-0.797** | 강한 음의 상관 (C17이 높으면 C65가 낮음) |
| **C10** | **0.478** | 중간 양의 상관 (C10이 높으면 C65도 높음) |
| **C39** | **0.478** | 중간 양의 상관 (C39가 높으면 C65도 높음) |
| **C12** | **0.338** | 약한 양의 상관 |

### 해석

- **C17이 가장 강력한 예측 변수**입니다. 상관계수 -0.797은 매우 높은 수준입니다.
- C10과 C39는 동일한 상관계수(0.478)를 가지며, 서로 밀접하게 관련된 변수일 가능성이 높습니다.
- 전체적으로 **강한 상관을 보이는 피처가 적은 편**이므로, 비선형 모델(예: LightGBM, XGBoost)이 선형 모델보다 더 나은 성능을 보일 가능성이 있습니다.

### 다중공선성

- C10과 C39가 매우 유사한 정보를 담고 있을 수 있으므로, 둘 중 하나를 제거하거나 주의가 필요합니다.

### 조치 방향

> C17, C10, C39, C12를 **핵심 피처**로 주목합니다.
> 트리 기반 모델(LightGBM 등)은 다중공선성에 강하므로, 우선은 모두 유지합니다.

## 8. PM(예방 정비) 영향 분석

### C33과 C65의 상관관계

- **피어슨 상관계수: -0.0130**

### 해석

- C33(PM 이후 경과 시간)과 C65(결함 수치)의 직선적 관계는 **거의 없습니다**.
- 단, 비선형적 관계(예: 정비 직후에는 좋다가, 일정 시간 후 급격히 나빠지는 패턴)가 있을 수 있으므로, 완전히 배제하기는 이릅니다.

### 조치 방향

> C33을 그대로 사용하는 것보다, **구간별 파생 변수**(예: PM 후 0~10시간, 10~50시간, 50시간 이상)로
> 변환하면 비선형 관계를 포착할 수 있습니다.

## 9. 이상치(극단값) 탐지

### C65 이상치

| 항목 | 값 |
|------|---|
| 정상 범위 | [-30.8, 1,839.2] |
| 이상치 개수 | **0개** / 11,939개 (0.00%) |

### 해석

- C65에는 IQR 기준 이상치가 **전혀 없습니다**.
- 정상 범위가 [-30.8, 1839.2]인데, 실제 데이터 범위(503~1678)가 이 안에 완전히 들어갑니다.
- 타깃 값이 매우 안정적이라는 뜻입니다.

### 조치 방향

> C65에 대한 이상치 처리(클리핑, 제거 등)는 **불필요**합니다.
> 센서 피처의 이상치는 트리 기반 모델에서는 자동으로 처리되므로, 별도 조치 없이 진행해도 됩니다.

## 종합 요약

### 데이터 품질: 우수

| 항목 | 평가 | 설명 |
|------|------|------|
| 결측값 | 양호 | 부분 결측 없음 |
| 분포 일관성 | 매우 양호 | Train/Valid/Test 분포 동일 (AUC ≈ 0.50) |
| 이상치 | 양호 | C65에 이상치 없음 |
| 타깃 분포 | 양호 | 대칭적, 변환 불필요 |

### 핵심 발견사항

1. **C17이 가장 강력한 예측 변수** (상관계수 -0.797)
2. 데이터셋 간 **분포 차이가 거의 없음** (Adversarial AUC ≈ 0.50)
3. 웨이퍼당 약 10개 행, **C65는 웨이퍼 단위 상수** → 웨이퍼 단위 집계 후 예측
4. **20개 상수 컬럼 제거 필요** (68개 → 48개)
5. PM(C33)의 직접적 영향은 미미 (r = -0.013)

### 권장 전처리 파이프라인

```
1. 상수 컬럼 20개 제거
2. 웨이퍼(C64) 단위로 센서값 집계 (mean, std, min, max 등)
3. 타깃(C65) 변환 불필요 (왜도 ≈ 0)
4. 결측값 처리 불필요
5. C33 구간화 파생 변수 생성 고려
6. C17, C10, C39, C12를 핵심 피처로 주목
```

### 권장 모델링 방향

- **트리 기반 모델 우선 시도**: LightGBM, XGBoost (비선형 관계 포착, 다중공선성에 강함)
- **교차 검증**: 랜덤 분할된 데이터이므로 일반적인 K-Fold CV 사용 가능
- **도메인 적응 불필요**: 분포가 동일하므로 표준 학습 파이프라인 적용
