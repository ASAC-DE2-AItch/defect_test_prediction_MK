# modeling_v12 — Lot 누수 검증 결과 (GroupKFold C64 vs C20)

> v5에서 확립한 "신규 Lot 정직 평가" 원칙(GroupKFold(C20=Lot))을 v12 FULL 결합 모델에 적용.
>
> 실험일: 2026-07-15

---

## 1. 배경

v12(FULL 192피처 결합 모델)는 wafer 단위 평가에서 valid RMSE 48.76 / test 49.15로
역대 최고 성능을 보고했다. 그러나 v5 누수 검증에서 밝혀졌듯 **valid/test는 train과 Lot을
99.9~100% 공유**하므로, 이 숫자는 '신규 Lot' 일반화 성능이 아니라 '같은 Lot 형제를 본'
낙관적 성능이다. v12는 지금까지 **GroupKFold(C20=Lot) 테스트를 거치지 않았다.** 본 검증은
동일 v12 피처·모델로 CV 그룹키만 C64(wafer) ↔ C20(Lot)으로 교체해 신규 Lot 성능을 측정한다.

- 데이터: train_data 123,614행 / wafer(C64) 11,939장 / Lot(C20) 1,217개 (Lot당 9.8장)
- 모델: v12 원본 그대로 — 192피처, LightGBM(lr=0.03, num_leaves=127), 5-fold GroupKFold 앙상블, C23 fold별 타깃인코딩
- C20(Lot)은 **검증 그룹키로만** 사용, 피처로는 계속 미사용(일반화 원칙 유지)

---

## 2. 검증 결과 — wafer CV vs Lot CV

동일한 v12 피처로 CV 그룹키만 교체:

| CV 방식 | OOF RMSE | R² | 의미 |
|---------|----------|-----|------|
| **GroupKFold(C64=wafer)** | **50.00** | 0.9635 | 현재 v12 방식. 같은 Lot 형제가 train에 존재 (낙관적) |
| **GroupKFold(C20=Lot)** | **72.06** | 0.9242 | 통째로 새로운 Lot 예측 (신규 Lot 실제 성능) |
| **낙관 편향** | **+22.05 pt** | — | Lot 누수로 부풀려진 크기 |

### fold별 상세

| fold | wafer RMSE | wafer best_iter | Lot RMSE | Lot best_iter |
|---|---|---|---|---|
| 0 | 51.11 | 434 | 66.85 | 171 |
| 1 | 49.55 | 410 | 76.20 | 114 |
| 2 | 48.88 | 747 | 77.02 | 131 |
| 3 | 51.17 | 674 | 71.34 | 143 |
| 4 | 49.25 | 447 | 68.30 | 86 |

- wafer CV OOF 50.00은 v12가 보고한 valid 48.76과 정합적(valid도 Lot 공유 → wafer CV와 동급).
- **Lot CV의 best_iter가 현저히 낮다(86~171 vs wafer 410~747).** Lot을 통째로 빼면 검증셋에서
  더 짜낼 'Lot 형제 신호'가 없어 early stopping이 일찍 걸린다 — 누수의 기계적 증거.

## 3. v5와의 비교 — 핵심 발견

| 모델 | wafer CV (같은 Lot) | Lot CV (신규 Lot) | 낙관 편향 |
|------|---------------------|-------------------|-----------|
| modeling_v5 | 62.63 (R² 0.943) | 70.42 (R² 0.928) | +7.79 pt |
| **modeling_v12 (FULL)** | **50.00 (R² 0.964)** | **72.06 (R² 0.924)** | **+22.05 pt** |
| 변화 (v5→v12) | **−12.63** (개선) | **+1.64** (악화) | +14.26 |

**v12의 wafer-CV 개선(−12.6pt)은 신규 Lot로 이어지지 않는다.** 오히려 신규 Lot 성능은
v5(70.42)보다 v12(72.06)가 소폭 나쁘다. 즉 v12 FULL의 화려한 피처(원본 센서 36 + WF 집계
144 등)가 **주로 Lot 형제 패턴을 엿보는 데 쓰였고**, 실제 신규 Lot에서는 v5 대비 이득이 없다.
낙관 편향이 +7.79 → +22.05pt로 커진 것이 이 과적합을 정량화한다.

---

## 4. 결론

1. **v12에 코드/구현 버그는 없다.** wafer fold 0 RMSE 51.108(best_iter 434)이 원본과 정확히 일치.
2. **v12의 신규 Lot 실제 성능은 RMSE ~72.** wafer 평가(50.0)는 신규 Lot 대비 +22pt 낙관적이다.
3. **v12 FULL의 개선은 대부분 Lot 누수 이득이다.** 신규 Lot 기준으로는 v5(70.4)와 사실상 동급
   내지 소폭 열세(72.1). "valid 48.76"을 실무 일반화 성능으로 해석하면 안 된다.
4. **재앙적 누수는 아니다.** 신규 Lot에서도 R² 0.92로 여전히 유효하다(v5 결론과 일관).
5. **두 성능을 분리 보고해야 한다:**
   - 대회 리더보드 (valid/test, Lot 공유): RMSE ~49~50
   - 실무 신규 Lot (Lot 단위 CV): RMSE ~72

## 5. 권고 (앞으로의 평가 원칙)

- **모델 선택 기준을 Lot CV로 전환.** wafer CV로는 v12가 v5를 크게 앞서지만(−12.6pt),
  신규 Lot 기준에선 우열이 사실상 사라진다. 리더보드가 아닌 실무가 목표라면 **v12 FULL이
  v5보다 낫다고 볼 근거가 없다.**
- **피처 다이어트/정규화 재검토.** WF 집계 144피처가 Lot 형제 지문(fingerprint)으로 작동할
  가능성이 높다. Lot CV를 목적함수로 두고 피처를 다시 선별하면 신규 Lot RMSE를 낮출 여지가 있다.
- **목표 RMSE 재설정.** 현재 목표(~40)는 wafer CV 기준이다. 신규 Lot 기준 ~72에서 개선을
  논해야 실무 의미가 있다.
- **전 버전(baseline~v13) Lot CV 재평가.** wafer CV 순위와 Lot CV 순위가 뒤바뀔 수 있으므로,
  "진짜 성능" 리더보드를 Lot CV로 다시 그릴 가치가 크다(v5 62.6→70.4, v12 50.0→72.1이 이미 역전 신호).

## 6. 방법 / 재현

- 스크립트: `lot_cv_test.py` (이 폴더). v12 `src/` 파이프라인을 그대로 임포트, CV 그룹키만 교체.
  - 계산 비용 절감을 위해 빌드 단계에서 **미사용 FDC 556열 피벗을 생략**하고 시간/레짐 피처만 생성
    (build_rows가 쓰는 입력은 동일 → 피처·결과 불변, fold0 재현으로 확인).
  - LightGBM 설정은 v12 원본(patience=100 포함) 그대로.
- 결과: `lot_cv_test_result.json`
- 실행:
  ```bash
  python lot_cv_test.py run wafer 0 1 2 3 4   # C64 그룹키 5-fold
  python lot_cv_test.py run lot   0 1 2 3 4   # C20 그룹키 5-fold
  python lot_cv_test.py agg                   # 집계 → JSON
  ```
