# modeling_v11 REPORT 01 — v10 − 시간/레짐 7개 (LotTE ablation)

> 스냅샷 고정: 2026-07-11 (세션). 이후 분석이 바뀌면 새 REPORT를 만든다(이 파일은 수정 안 함).
> 모델: `modeling_v11.ipynb` — v10에서 시간/레짐 7개 제거 = 186피처. 나머지 파이프라인은 v10과 동일.

---

## 1. 한 줄 요약

v10에서 **시간/레짐 7개**(`is_post_loud_pm`, `days_since_last_pm`, `hour`, `dslp_x_hour`,
`hour_x_c33`, `post_pm_days`, `is_special_recipe`)만 제거한 ablation. 결과 **valid 34.70 / test 35.10**
로 **v10보다 소폭 악화**(valid +0.41, test +0.77) → **시간피처는 빼면 손해**임을 확인.

## 2. 성능 (v10 대비)

| 버전 | 피처 | OOF | valid | test |
|------|------|-----|-------|------|
| v10 (combined + Lot TE) | 193 | 34.87 | **34.29** | **34.33** |
| **v11 (v10 − 시간7)** | 186 | 35.63 | **34.70** | **35.10** |
| Δ (v11 − v10) | −7 | +0.76 | +0.41 | +0.77 |

## 3. 오차 분석 (valid, n=1990)

- RMSE 34.70 · MAE 24.73 · bias +0.41 (무편향).
- 5분위 RMSE (낮은 y→높은 y): 22.6 / 29.5 / 26.8 / 33.9 / **52.7** — v10과 동일하게 **고불량 tail 과소예측**.
- 최대오차 3건 모두 과소예측: y=1666→1141(+525), y=1221→1053(+168), y=1343→1180(+163).
- v10 대비 전 구간이 소폭씩 나빠짐(특정 구간 손실이 아니라 **전반적 소폭 열화**) → 시간피처가 국소가 아닌 전역적으로 미세 기여.

## 4. 해석

- Lot 타깃인코딩(`C20_te`)이 점수를 지배하는 상황에서도, 시간/레짐 7개는 **여전히 valid −0.4 / test −0.8pt의 실질 기여**를 함.
- combined 원 개발 시 시간피처가 valid를 61→48로 크게 낮췄던 것과 방향 일치(기여폭은 Lot TE에 상당부분 흡수돼 축소).
- **결론: 대회 점수 관점에서 시간피처는 유지가 유리. v10 채택, v11은 제거 시 손해를 수치로 남기는 ablation 근거.**

## 5. 누수 주의 (v10과 동일)

`C20_te`는 valid/test가 train과 Lot을 공유해 나오는 누수성 점수. 신규 Lot(GroupKFold C20)엔 무너짐.
대회 전용 계열이며 프로젝트 원칙(Lot 피처 금지)과 배치됨.
