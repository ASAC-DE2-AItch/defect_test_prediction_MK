## v1 대비 변경사항

### 추가된 피처 그룹 4가지

| # | 피처 그룹 | 명명 규칙 | 예시 | 목적 |
|---|----------|----------|------|------|
| 1 | **Step별 센서 집계** | `{센서}_step{번호}_{통계}` | `C17_step4_mean` | 각 공정 단계에서의 센서 상태를 개별 포착 |
| 2 | **Step 간 전이** | `{센서}_trans_{from}to{to}` | `C17_trans_1to4` | 인접 step 간 센서 변화량 |
| 3 | **교호작용** | `{센서1}_x_{센서2}`, `_div_` | `C17_x_C12` | 핵심 센서 간 곱/비율 |
| 4 | **C33 구간화** | `C33_bin` | 0~3 | PM 경과시간을 4구간으로 범주화 |

### 피처 수 변화

- v1: **315개** (글로벌 집계만)
- v2: **816개** (글로벌 + Step별 + 전이 + 교호작용 + 구간화)

### 변경하지 않은 것

- 모델: LightGBM (동일 하이퍼파라미터)
- CV: GroupKFold 5-Fold (C64 기준)
- 전처리: 동일한 21개 컬럼 제거

## 용어 정리

| 용어 | 설명 |
|------|------|
| Step별 집계 | C7(공정 단계)가 1,4,5,6,7 각각일 때의 센서 통계를 별도로 계산 |
| 전이 피처 | step1→step4처럼 인접 단계 간 센서 평균의 차이 |
| 교호작용 | 두 센서의 곱(×) 또는 비율(÷)로 만든 파생 변수 |
| C33 구간화 | PM 후 경과시간을 0~10h, 10~50h, 50~100h, 100h+ 4구간으로 나눔 |
| OOF | Out-Of-Fold, 교차검증에서 각 fold의 검증 데이터에 대한 예측 |
