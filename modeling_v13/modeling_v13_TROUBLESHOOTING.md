## 2. 핵심 트러블 & 교훈 (인과사슬)

### T1. 축 오선택 — GKF(lot-CV)로 골랐더니 시간 드리프트를 못 잡음
- **문제**: M3~M5의 모든 선택·튜닝을 GroupKFold(C20)로 판정. GKF는 lot을 **무작위** 분할이라 학습·검증이 **시간이 뒤섞임** → 학습이 미래 시기(고레짐)를 이미 봐서 그 레짐을 "외움".
- **결과**: 챔피언 −4.6이 **가짜**였다. 미래 Lot(B2)에선 66.7→77~105.
- **교훈**: **GKF는 필요조건이지 충분조건이 아니다.** Lot 암기(KFold보다 정직)는 잡아도 **시간 드리프트는 못 잡는다.** 배포가 "미래 Lot 예측"이면 **처음부터 시간축으로 판정**해야 한다.

### T2. 센서 운영 무가치 — 5중 확증
- M4.5 잔차–센서 corr **0.037** → ablation 센서 89개가 core10 단독(78.2)보다 나쁨 → REPORT_11 센서–타깃 **부호반전** → REPORT_12 드리프트 프록시 lot-CV **무가치** → **B2′ 재학습해도 센서<core10**.
- **교훈**: v13의 센서 복원(C11·C15·C16·C31)·655풀·GA·챔피언 전부 **운영축에서 미실현.** 웨이퍼-집계 FDC는 백본(시간/레짐/PM) 이상의 미래 신호를 담지 못한다.

### T3. 비정상성의 정체 = PM 레짐 시프트
- 첫 major PM(**2018-12-24**)에서 C65 평균 **638±40 → 1217±221**(1.9배·스프레드 5.5배). is_high_regime이 이걸 인코딩하나, 고레짐 내 Lot 분산(±221)은 못 맞춤(M4.5 천장의 시간판).
- **교훈**: 남은 오차는 **Lot 고유 + 시간 드리프트** 양축. 센서로 접근 불가.

### T4. C25 함정 — 최상위 기여 피처가 시간에 뒤집힘
- B4에서 챔피언 gain 최상위가 **C25(노후도 드리프트, 달력 corr −0.30)**. 그런데 C25의 타깃 corr가 학습기 내 **−0.10~−0.19 → +0.02~+0.04로 부호반전**. lean-85가 제거한 14개 중 **9개가 C25**.
- **교훈**: 중요도 높은 피처일수록 **시간 안정성**을 의심하라. 노후/드리프트 프록시는 예측력처럼 보여도 미래엔 독.

### T5. 홀드아웃 예산 소진 — 그리고 fresh future가 없음
- R5(후기 구간 조회 통산 2회): M6 B2(1회)·M8 B2′(1회)로 **소진**.
- **valid_X는 train과 같은 시기(2018-12~2019-02 동일)** = *same-era 독립 샘플*이지 **미래 홀드아웃이 아님.** test_X는 C40 파싱 불가. → **깨끗한 미래-시간 홀드아웃이 이 데이터셋엔 없다.**
- **교훈**: 시간축 검증 자원은 유한하다. v14는 **학습기 내부 nested 시간-CV**로 개발하고, 최종 확인의 한계를 정직하게 안고 가야 한다.

### T6. 테스트 설계 함정 — PM 경계 straddle
- F2′ 시간견고성 테스트(early→late Dec)가 **PM 경계를 가로질러** early=저레짐/late=고레짐 → 전 셋 ~613(미학습 레짐레벨 외삽)으로 **무효**. 피처셋 비교 불가.
- **교훈**: 시간 테스트는 **레짐 경계를 인지**해서 설계해야 한다(레짐 내부 분할 또는 재학습 백테스트). 경계를 무심코 넘으면 레벨 외삽이 신호를 덮는다.

## 3. 방법론 교훈 (일반화)

1. **정직-CV의 승리**: lot-CV나 valid 점수만 봤으면 챔피언(−4.6)을 "성공"으로 배포 → 현장에서 core10보다 나쁜 채로 붕괴했을 것. **시간축(B2/B2′)이 배포 전에 잡았다.**
2. **사전등록의 값**: 게이트·문턱을 결과 전에 못박아 체리피킹을 원천 차단. 실패도 "정직 종료"로 깨끗이 매듭.
3. **천장은 데이터다**: 파라미터(M5 −4.6)로 gap-A는 닫았으나 M4.5 천장(Lot·시간)은 못 뚫음 — 이건 모델이 아니라 데이터/공정의 성질.
4. **재학습은 정공법**: 비정상 공정에선 무재학습이 배포 불가(255), 재학습이 2.5배 회복(100). 단 그래도 R² 0.85 — 센서 없이 백본으로.

## 4. v14로 넘기는 것 (무엇을 다르게)

- **① 축 교정**: 시간축(rolling/nested time-CV)을 **처음부터** 의사결정 축으로. lot-CV·valid는 보조·기록.
- **② 드리프트-겨냥 신규 피처**: 기존 집계 재선별(=lean-85, 이미 실패)이 아니라 **설계상 정상성을 갖는 새 피처** — PM-상대·레짐-조건부·최근 baseline 대비 편차·타깃 레벨/잔차 분해.
- **③ 홀드아웃 현실**: fresh future 없음 → 학습기 내부 nested 시간-CV 개발 + valid/test는 same-era 확인용. 최종 시간 확인의 한계 명시.
- **④ 중단 기준 선언**: 천장이 낮다는 걸 알고 시작 — "N회 무개선 시 정직 종료" 사전 못박기.
