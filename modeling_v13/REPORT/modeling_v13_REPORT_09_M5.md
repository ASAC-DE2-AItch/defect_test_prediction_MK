## 4. 정직한 발견 (전략 함의)

1. **대형 개선의 정체 = gap-A(파라미터) 해소, Lot 천장 돌파 아님.** M8_PARAMS(num_leaves=175, 705r)는 10피처용 → 99피처에서 **KFold 50.6 ↔ GKF 71.4 = 20.8pt lot-mate 과적합**. 튜닝셋은 규제 강함(LGBM num_leaves 50·min_child 80·bagging 0.55 / **XGB max_depth 4·gamma 2.35·강 reg**, 246r) → 신규-Lot 견고. **M4.5가 지목한 "파라미터 1순위" 지렛대가 −4.6 실현.** M4.5의 잔차-센서 0.037(천장)은 **여전히 유효** — 이 개선은 센서/Lot 신호가 아니라 **백본 파라미터를 정직축에 정렬**한 것. 모순 아님.
2. **XGBoost 확정 = v8 M7 미결('ET 우위=암기?')의 최종 답.** 정직축: **XGB(반-암기) > LGBM > ET(암기)**. REPORT_06(−0.6)·REPORT_07(다중시드 −0.59) 사전예측과 일치, 이제 튜닝 후에도 유지.
3. **in-selection 낙관 통제됨.** stable_GKF(66.743)는 Optuna가 최적화한 폴드라 낙관편향 내포하나, **독립 다중시드 seed_mean(66.978)과 격차 +0.235로 작음** → 폴드맵 과적합 아님. 단 세 시드 모두 **같은 1217 lot 공유** → 진짜 외삽(미래 Lot)은 **B2 시간분할이 최종 심판**.
4. **기대 초과 — 정직 기록.** 강건 사전 예측("80 trials면 gap-A 거의 닫혀, 개선 modest")보다 큼. 원인 = **M8_PARAMS의 99셋 과적합 폭을 과소평가**. 방향(파라미터가 지렛대)은 M4.5대로 옳았음. 결과를 본 뒤 규칙 변경 없음(R4).

### A.1 M5가 무슨 과정인가 — 무엇을 · 왜 · 무엇을 원해서

**한 문장**: "재료는 그대로 두고, 요리법만 이 재료에 맞게 다시 잡는" 단계.

비유로 보면:
- **피처(입력 데이터)** = 요리 **재료**. M4까지 99개로 확정(Cons-GA 99). **M5는 이걸 안 건드린다.**
- **모델** = 요리사. **하이퍼파라미터** = 요리 **방식**(불 세기·조리 시간·간). M5는 이 방식만 최적으로 조정한다.

- **무엇을**: 확정된 99개 피처를 고정하고, 모델 설정값(하이퍼파라미터)을 자동 탐색기(Optuna)로 수백 조합 시험해 가장 좋은 걸 찾는다. LGBM·XGBoost 두 모델 각각.
- **왜**: 기존 기준 모델은 **"피처 10개 시절에 맞춰 둔 설정"** 을 99개에 그대로 쓰고 있었다 → 몸에 안 맞는 옷. 이걸 99개용으로 다시 재단하는 것.
- **무엇을 원해서(목표)**: **학습에 없던 새 Lot**(처음 보는 웨이퍼 묶음)의 예측 오차를 기준선보다 최소 0.5 낮추기(게이트 G5).
  - 왜 하필 '새 Lot'? 현업에선 늘 처음 보는 Lot을 예측한다. 과거 Lot을 통째 외운 모델은 현장에서 무너진다. 그래서 "안 본 Lot"에서의 성능(GroupKFold)만 점수로 인정한다.

### A.2 결과 쉽게 읽기

- **RMSE(오차)** = 예측이 실제에서 평균 얼마나 빗나가는지. C65와 같은 단위이고 **낮을수록 정확**.
- 기준선 **71.4 → 챔피언(XGBoost) 66.7**. 약 **4.6 낮아짐** = 목표(0.5)의 **9배** 초과 달성.
- **R² 0.935** = 타깃 변동의 93.5%를 설명(0.9 이상 유지 조건도 통과).
- 두 모델 중 **XGBoost 승**. 데이터를 나누는 방식(시드)을 3가지로 바꿔 재확인해도 일관되게 좋았다 → **운이 아니라 실력**.

### A.3 어떤 파라미터를, 왜 바꿨더니 낮아졌나 (핵심)

한마디로 **"모델을 덜 복잡하게 · 덜 외우게" 만든 것**뿐이다. 기준 모델(M8_PARAMS)은 99피처에 비해 과하게 공격적이라 **훈련 Lot의 우연한 특징까지 통째 암기** → 새 Lot에서 무너졌다. 튜닝은 그 반대로 갔다.

| 파라미터 | 기준(baseline) | 튜닝 후 | 쉬운 의미 | 왜 오차가 내려갔나 |
|---|---|---|---|---|
| 트리 복잡도 (`num_leaves`/`max_depth`) | num_leaves **175** | LGBM **50** · XGB depth **4**(매우 얕음) | 트리를 얼마나 잘게 쪼개 규칙을 만드나 | **가장 큰 축.** 175 = 자잘한 패턴까지 암기→새 Lot에 안 맞음. 얕게 = 큰 그림만 학습→일반화 |
| 분기 최소 표본 (`min_child_samples`/`_weight`) | **5** | LGBM **80** · XGB **~7** | 규칙 하나 만들 때 최소 몇 개 데이터가 필요한가 | 5 = 데이터 몇 개로 규칙화(노이즈까지)→과적합. 80 = 우연한 패턴 배제 |
| 학습 속도 (`learning_rate`) | 0.029 | LGBM **0.0095** | 한 번에 얼마나 급하게 배우나 | 낮게 = 조금씩 신중히→과적합 완화 |
| 무작위 샘플링 (`feature_fraction`/`colsample`/`subsample`) | 거의 전량 | **0.55~0.88** | 트리마다 데이터·피처의 **일부만** 무작위로 봄 | 매번 다른 부분만 학습→특정 신호 과의존 방지(규제). *※ 피처를 지우는 게 아니라 매번 랜덤 표집* |
| 분기 억제 (`gamma`/`lambda`) | 약함 | XGB gamma **2.35**·강 reg | "이만큼 이득 없으면 가지 안 침"+복잡도 벌점 | 불필요한 세밀 분기 차단 |
| 학습 횟수 (rounds) | 705 | XGB **246**(조기중단) | 너무 오래 학습하면 과적합 | 성능 꺾이는 지점에서 멈춤 |

**공통 테마 = 규제(regularization)**: 여섯 개 전부 모델을 *얌전하게* 만드는 방향이고, 방향은 하나다 — **훈련 Lot 암기 ↓, 새 Lot 일반화 ↑.**

**증거(암기→일반화 전환)**: 기준 모델은 과거 Lot이 섞인 채점(KFold)에선 **50.6**인데 새 Lot 채점(GroupKFold)에선 **71.4** — 이 **20.8 격차가 곧 "과거 Lot 암기량"** 이다. 튜닝은 이 격차를 줄여(덜 외우게 해서) 새 Lot 점수를 끌어내렸다.

### A.4 그래서 — 피처는 불변, 천장도 불변

- **재료(피처 99개)는 1도 안 바뀌었다.** 바뀐 건 요리법(모델 설정)뿐.
- 개선 −4.6의 정체: **① 파라미터 재튜닝(같은 LGBM만으로 −3.7) + ② 모델 교체(→XGB, 추가 −0.9).** 둘 다 피처가 아니라 **모델 쪽**.
- 그래서 **M4.5에서 진단한 "천장"은 그대로다.** M4.5는 "남은 오차는 센서로 설명 안 되는 Lot 고유 효과(상관 0.037)라 피처로는 더 못 내린다"고 했는데 — M5는 그 천장을 **뚫은 게 아니라**, 기준 모델이 낭비하던 **"설정 미스매치" 손실을 회수**한 것이다.
- 함의: 앞으로 피처로 더 짜낼 여지가 적다는 M4.5 결론은 유효. 이번 이득은 **파라미터라는 별개 지렛대**에서 나왔다. (다음 관문 M6은 이 −4.6이 진짜 미래 Lot에도 버티는지 확인하는 단계.)
