## 0. 무엇을 보는가

M-T 대조군이 "센서(C59·C60 step4 집계 + 레시피)는 시간축 위에서 +3.91pt 기여한다"를 보였다. M2는 한 걸음 더 나아가 묻는다 — **코어 10에 쓴 센서 3종 말고, 나머지 FDC 집계 풀(23센서×5통계×step ≈ 550개)에 추가로 건질 신호가 있는가?** 특히 EDA(E7)에서 급등 레짐 내부 상관 +0.459였던 **C25**가 gain 랭킹에 올라오는지가 관전 포인트다.

---

## 1. 방법 — 2-pass gain 재선별 (FS §6)

- **풀**: 코어 10의 비센서 8개(시간/레짐 7 + `is_special_recipe`) + **전체 센서 집계 풀** = **563 피처**. (id·타깃·문자열 `C23`/`C6`·pkl 별칭 `is_post_pm`/`post_pm_days` 제외.)
- **pass 1**: 풀 전체로 wafer 5-fold OOF 학습 → fold-평균 **gain** 랭킹 산출.
- **pass 2**: gain 상위 **TOP_N ∈ {10,12,15,20,25}** 재선별 후 각각 CV.
- **보조 probe**: 검증된 코어 10을 **유지**하고 gain 상위 신규 센서 K∈{3,5,8}개만 **추가**.
- 파라미터·CV 스킴은 M0~M1과 동일(M8_PARAMS, wafer `KFold(shuffle,42)`). valid/test는 최고 조합 보고용(선택엔 미사용).

## 2b. 보조 probe 상세 — 어떤 신규 센서가 추가됐나

보조 probe는 코어 10을 그대로 두고 **gain 상위 신규 센서(코어 10 밖)를 누적으로** 추가한 것이다. 추가 순서·gain(전체 풀 5-fold 평균 gain 상대%)과 각 probe 포함 여부:

| 신규 순위 | 센서 | gain % | 전체 gain 순위 | +3 | +5 | +8 |
|:---:|---|:---:|:---:|:---:|:---:|:---:|
| 1 | `C12_mean_step6` | 4.38% | 4 | ✓ | ✓ | ✓ |
| 2 | `C12_mean_step7` | 0.46% | 7 | ✓ | ✓ | ✓ |
| 3 | `C4_mean_step4` | 0.33% | 9 | ✓ | ✓ | ✓ |
| 4 | `C4_max_step1` | 0.26% | 10 | | ✓ | ✓ |
| 5 | `C12_max_step6` | 0.21% | 12 | | ✓ | ✓ |
| 6 | `C61_mean_step1` | 0.09% | 13 | | | ✓ |
| 7 | `C60_std_step4` | 0.07% | 14 | | | ✓ |
| 8 | `C62_mean_step1` | 0.07% | 15 | | | ✓ |

**probe별 결과** (추가 센서는 이전 단계 대비 증분):

| probe | 이번 단계 추가 센서 | CV | ΔvsM0b |
|---|---|:---:|:---:|
| 코어10 + 3 | `C12_mean_step6`, `C12_mean_step7`, `C4_mean_step4` | 41.05 | +0.71 |
| 코어10 + 5 | (+3) + `C4_max_step1`, `C12_max_step6` | 41.29 | +0.95 |
| 코어10 + 8 | (+5) + `C61_mean_step1`, `C60_std_step4`, `C62_mean_step1` | 43.03 | +2.69 |
| 코어10 + C25 | `C25_std_step4`(23위), `C25_mean_step4`(27위) | 43.66 | +3.31 |

> **패턴**: 신규 센서 8개 중 **C12가 3개**(step6·step7 평균, step6 최대)·**C4가 2개**로 편중된다. `C12_mean_step6`은 신규 중 유일하게 gain 4.38%로 튀지만 추가 시 오히려 악화 — EDA에서 C12는 전체 상관 0.338이나 **레짐 내부 −0.02/−0.10으로 죽는 레짐 프록시**(C17과 동형)라, 그 gain은 레짐 신호에서 빌려온 것이고 실제 투입 땐 중복·노이즈만 얹힌다. C25는 23·27위(0.05~0.06%)로 부상 자체를 못 한다. → **"gain 높아 보이는 신규 센서"조차 실익이 없음**이 확증된다.

## 3. 해석 — 코어 10은 이미 (준)최적, gain 선택은 오히려 독

세 가지가 드러났다.

**① C59/C60의 "집단 필수성" 실증.** 코어의 `C59/C60_mean_step4`는 개별 gain이 18위권으로 미미하지만, M-T 대조에서 이 3종을 빼면 CV가 +3.91pt 나빠졌다(집단으로 필수). **gain 기반 TOP_N은 이 저-gain·고-집단가치 피처를 떨궈버리고**, 대신 개별 gain은 높지만 일반화가 나쁜 센서(C12_mean_step6 등)를 주워 CV를 악화시킨다. FEATURE_SPEC §5의 경고("개별 gain 낮아도 유지")가 정확히 재현됐다.

**② C25 신화의 종결.** EDA의 급등 내부 +0.459는 **레짐을 통제하기 전** 상관이었다. 레짐 피처(`is_high_regime`+시간)가 이미 그 변동을 흡수하므로, 모델 안에서 C25 aggregate는 gain 23위로 밀리고 추가해도 CV가 +3.31pt 악화된다. **C25는 "레짐 프록시성 상관"이었지 독립적 예측 신호가 아니었다** — M1의 C23_te와 같은 패턴.

**③ 풀 통째 투입은 노이즈.** 563피처 전체 CV 53.01은 소표본(11,939 WF)에서 고차원 센서 풀이 과적합함을 보인다. v1~v7이 센서 풀에 의존해 ~60에 갇힌 구조와 동일 — v8의 **"시간/레짐 소수 피처 + 검증된 센서 3종"** 미니멀리즘이 옳다는 재확인.

> 결론: **코어 10 유지.** 센서 확장은 실익이 없고, 오히려 최종 피처셋을 키우면 M4에서 손해다. M-T가 보인 센서 기여(+3.91pt)는 이미 코어 안의 C59/C60/레시피가 전부 담고 있다.
