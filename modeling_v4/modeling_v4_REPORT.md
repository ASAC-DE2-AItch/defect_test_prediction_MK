## 4. 실패 분석

### 4.1 왜 악화되었나

1. **노이즈 피처 폭증**: 센서 교차 피처만 360개. 16센서 중 대부분 쌍은 C65와 무관 → 노이즈
2. **FFT의 한계**: WF당 ~10행(step)으로 시계열이 너무 짧아 FFT가 의미 있는 주파수를 추출하지 못함
3. **하이퍼파라미터 불일치**: v3 Optuna 파라미터가 299개 피처에 최적화된 것. 901개 피처에 그대로 적용하니 `colsample_bytree=0.655` → ~590개 피처 사용 → 노이즈 과다 유입
4. **피처 그룹 분석 버그**: `.values`로 학습시켜 컬럼명 소실 → 어떤 그룹이 기여했는지 확인 불가

### 4.2 근본 문제 진단

v1~v4 실험을 통해 확인된 것:

| 시도 | 결과 | 시사점 |
|------|------|--------|
| v1: 기본 통계 집계 | RMSE 62.53 | 기본 접근법의 성능 |
| v2: Step별 분리 집계 | +0.19 악화 | Step당 행이 1~2개라 분리 효과 없음 |
| v3: Optuna 튜닝 | -0.22 개선 | 하이퍼파라미터 여유분은 0.2pt 수준 |
| v4: 다양한 파생 피처 | +0.89 악화 | 무차별적 피처 추가는 노이즈만 증가 |

**결론: "WF 단위 통계 집계 → 테이블 기반 모델" 프레임워크 자체가 RMSE ~62의 천장을 가짐.**

## 5. RMSE 40을 위한 패러다임 전환 필요

현재까지의 접근법은 "row-level 데이터를 WF 단위로 집계한 뒤 tabular ML"입니다. 이 프레임 내에서는 피처를 어떻게 바꿔도 ~62 벽을 못 깹니다.

### 가능한 패러다임 전환 방향

| 방향 | 핵심 아이디어 | 기대 | 난이도 |
|------|-------------|------|--------|
| **A. Row-level 예측** | row(step) 단위로 C65를 예측한 뒤 WF 내 평균 | 개별 step 정보 보존 | 중 |
| **B. Sequence 모델** | WF 내 step 시퀀스를 1D-CNN/LSTM으로 학습 | 시계열 패턴 직접 학습 | 상 |
| **C. 데이터 재탐색** | 상수로 판단해 제거한 컬럼, 또는 C23/C30/C36 등 미사용 컬럼 재검토 | 누락 정보 발견 가능 | 하 |
| **D. 멀티모델 앙상블** | LGB+XGB+CatBoost → 최적 가중 평균 | ~2pt 개선 기대 | 중 |
