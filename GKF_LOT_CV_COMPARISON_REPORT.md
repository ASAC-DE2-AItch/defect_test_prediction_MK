## 0. 한 줄 요약

세 버전을 정직축 **GKF(C20=Lot)** 로 정렬하면 wafer 평가 순위가 **정반대로 뒤집힌다**. wafer CV 최강 v12 FULL(50.0)이 정직축 최하위권(72.06), 슬림한 **v13 Cons-GA(99)+XGBoost(70.77, 시드평균 70.75)가 현재 최적**. 낙관편향(암기 갭)은 피처가 화려할수록 커지고(v5 +7.8 → v12 +22.1), 같은 피처셋 안에서는 XGB가 가장 작다(+18.5). 다음 레버리지는 새 센서가 아니라 **core10(레짐/시간) 확장**이며, 크리티컬 패스는 **M5 튜닝 실행**이다.

## 1. 왜 GKF(C20)인가 — 누수 검증의 계보

### 1.1 v5 — 최초 검증 (멘토 지적, 원칙의 출발점)

**왜 했나**: 멘토가 "v5 RMSE ~60이 비정상적으로 좋다, 누수 아니냐"고 지적한 것에 대한 검증.

**메커니즘**: Lot ID(C20)를 피처로는 쓰지 않았지만, **CV를 wafer(C64) 단위로 쪼갠 것**이 문제였다. 검증 wafer의 99.9%가 train에 같은 Lot 형제를 갖고 있어, 모델이 센서 유사성으로 같은 Lot 패턴을 미리 엿본 셈(**간접 Lot 누수**). valid/test도 train과 Lot을 99.9~100% 공유하므로 valid 검증조차 '신규 Lot' 테스트가 아니었다.

**결과 — 동일 v5 피처, CV 그룹키만 교체**:

| CV 방식 | OOF RMSE | R² | 의미 |
|---|---|---|---|
| GroupKFold(C64=wafer) | 62.63 | 0.943 | 같은 Lot 형제 존재 → 낙관적 |
| GroupKFold(C20=Lot) | 70.42 | 0.928 | 통째로 새로운 Lot 예측 → 신규 Lot 실제 성능 |
| **낙관 편향** | **+7.79pt** | — | Lot 누수로 부풀려진 크기 |

**결론**:
- v5 코드/구현 버그 없음 — GroupKFold·타깃인코딩·ID 배제 모두 올바름.
- 멘토 우려는 부분적으로 타당 — wafer 평가가 신규 Lot 대비 ~8pt 낙관적.
- **재앙적 누수는 아님** — Lot ID를 직접 쓰면 RMSE 30(R² 0.99)까지 떨어지는 것과 달리, 신규 Lot에서도 R² 0.93으로 여전히 유효.
- **성능 분리 보고 원칙 확립**: 리더보드(valid/test, Lot 공유) ~60~62 / 실무 신규 Lot(Lot CV) ~70.

### 1.2 v12 — FULL 결합 모델 재검증 (편향의 확대)

동일 실험을 v12 FULL(192피처, wafer valid 48.76 '역대 최고')에 적용:

| CV 방식 | OOF RMSE | R² | 의미 |
|---|---|---|---|
| GroupKFold(C64=wafer) | 50.00 | 0.9635 | 낙관적 |
| GroupKFold(C20=Lot) | 72.06 | 0.9242 | 신규 Lot 실제 성능 |
| **낙관 편향** | **+22.05pt** | — | v5 대비 약 3배 |

- **기계적 증거**: Lot CV의 best_iter가 현저히 낮음(86~171 vs wafer 410~747) — Lot을 통째로 빼면 짜낼 'Lot 형제 신호'가 없어 early stopping이 일찍 걸림.
- **핵심 발견**: v12의 wafer-CV 개선 −12.6pt(v5 대비)가 **신규 Lot로 전혀 전이되지 않음**. 오히려 신규 Lot에선 v5(70.42)보다 소폭 열세(72.06). FULL의 화려한 피처(원본 센서 36 + WF 집계 144)가 주로 **Lot 형제 지문(fingerprint) 암기**에 쓰였다는 뜻.

### 1.3 v13 — 검증 도구에서 의사결정 축으로 승격

v5·v12에서 '검증용'이던 GKF(C20)를 v13은 **유일한 의사결정 축(현업축)** 으로 채택:

- KFold(`fold_kf5`)·valid/test는 **기록만, 의사결정 금지**(R1).
- `stable_group_kfold` 동결 — 표준 sklearn GroupKFold의 환경 의존(numpy argsort 타이브레이크) 제거, 어느 환경에서도 Δ0.000 재현.
- 다중시드 lot-CV(seed 1/2/3)를 보조축으로 — 단일 폴드맵 요행 배제.
- 이 축 위에서 피처 다이어트(655 풀 → Cons-GA 99), 4모델 맞대결, core10 ablation 수행.

## 3. 낙관편향 = 암기 갭 — 견고성 지표로서의 재해석

같은 Lot 형제를 허용하는 평가(wafer CV/KFold)와 정직축(Lot GKF)의 차이를 "암기 갭"으로 정의하면:

| 구성 | wafer/KFold ① | Lot GKF | **암기 갭** |
|---|---|---|---|
| v5 | 62.63 | 70.42 | **+7.79** |
| v12 FULL (192) | 50.00 | 72.06 | **+22.05** |
| v13 ET (99) | 48.48 | 72.42 | **+23.93** |
| v13 LGBM (99) | 50.68 | 71.37 | **+20.69** |
| v13 XGBoost (99) | 52.26 | 70.77 | **+18.51** |

① v5·v12는 GroupKFold(C64=wafer), v13은 wafer 단위 KFold(`fold_kf5`) — 개념상 동일(같은 Lot 형제 허용 평가).

**세 가지 패턴**:

1. **갭은 피처 용량과 함께 커진다** — v5(+7.8) → 192피처 FULL(+22.1). 피처가 화려할수록 Lot 형제 암기에 쓰인다.
2. **같은 99피처에서도 모델별 갭이 다르다** — ET(+23.9) 최대 = lot-mate 암기 최다·정직축 최약체, XGBoost(+18.5) 최소 = 신규 Lot 견고. 갭 크기와 정직축 순위가 정확히 역순.
3. **평가축 선택이 결과를 가른다** — KFold 순위(ET 최우)와 GKF 순위(XGB 최우)가 정반대. KFold로 모델을 골랐다면 최악의 정직축 모델을 뽑고 최선을 버렸을 것.

## 4. 판정 — 어떤 방식이 최적인가

**Cons-GA(99) + XGBoost** (M5 튜닝 확정 전 잠정 챔피언).

- **다중시드 검증 통과 (REPORT_07)**: stable + seed 1/2/3 **4파티션 전승**. Δ(LGBM−XGB) 평균 +0.590 · 최악 +0.426 → §6.2 기준(평균 ≥0.5 ∧ 최악 ≥0) 통과 → M5 챔피언 후보 편입(PLAN v2.4).
- **반-암기 프로파일**: KFold 열세 + GKF 우위 (ET와 정반대) — 우위가 암기가 아닌 일반화에서 나옴.
- **유보 1 — 미튜닝 대결**: LGBM은 M8_PARAMS(num_leaves=175, 공격적)라 과적합 핸디캡이 섞였을 수 있음 → **M5 GKF 동시 튜닝(LGBM·XGB)이 최종 심판**.
- **유보 2 — 버전 드리프트**: XGB 절대값이 라이브러리 버전에 ±0.26 의존(70.773 ↔ 70.517 실측) → **P0-3 xgboost 버전 핀이 공식화의 선행 조건**.
- **게이트 G5(≤70.712)**: XGB 원값 70.75~70.77로 **0.03~0.06 미달** — 튜닝으로 넘을 사정권. 단 판정은 M5 튜닝 챔피언 + 다중시드 + 버전 핀으로만.

방식 차원의 교훈: **"피처 슬림(99) + 정규화 강한 모델 + Lot 단위 정직 CV"** 조합이 GKF에서 이긴다. wafer 평가로 최적화한 v12 FULL식 피처 확장은 정직축에선 역효과였다.

## 5. 피처 디벨롭 방향 — 무엇을 기준으로 팔 것인가

core10 ablation(REPORT_08)이 우선순위를 정량으로 정해줬다:

1. **core10(레짐/시간) 확장 — 1순위.** 10개 백본 제거만으로 정직축이 +7.9~8.4pt 무너짐(71.4→79.3) = 프로젝트 최대 단일 기여 블록. C65 분산의 98.7%가 Lot 간(M4.5)인데 Lot을 설명하는 물리적 대리변수가 곧 시간/PM/레짐이다. **PM 경과·누적 파생, 레짐 전환 경계 지표, C33 drift의 비선형 변환** 등 이 축의 확장이 기대이득 최대.
2. **센서는 개수 확장 금지, core10과의 상호작용으로.** 센서 89개 전부(79.3)가 core10 단독(78.2)보다도 나쁨 — 센서는 단독 예측력이 아니라 **백본 위 상보 신호**(core10 78.2 → +센서 71.4, −6.8pt). 추가한다면 **레짐 조건부 센서 집계**(레짐별 mean/last, PM 전후 분리 통계) 형태.
3. **집계 통계 다양화 — 소규모 프로브만.** 잔차 Lot-ICC 0.784로 Lot 레벨 신호가 남아 있으나, 잔차–센서 최대상관 0.037(M4.5) — 현재 mean/std/max/min/last 형태로는 접근 난망. skew/quantile/slope 같은 분포·형태 통계가 유일한 우회로지만 기대치는 낮게.
4. **후순위 확정**: missing-indicator(F3′) 기대이득 낮음(REPORT_06), SegLGBM 무이득, ET는 F5′ 앙상블 재료로만 동결 보존.
