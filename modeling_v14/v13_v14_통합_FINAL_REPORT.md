## 4. 정직한 성과

1. **재학습은 필수이자 효과적.** 무재학습은 배포 불가(pooled 254.9, 147→319 단조 발산). 재학습이 2.5배 회복(→98.4). 비정상 공정의 정공법이 재학습임을 실증.
2. **FDC 센서의 운영 무가치를 이중 확증.** v13: 재학습해도 lean-85(99.84)가 core10(98.36)을 못 이김. v14: 센서를 시간-정상 상대값으로 바꾼 P1~P4조차 전부 악화 — 잔차–센서 상관 0.037 벽(M4.5)을 시간축에서 독립 재확인.
3. **정확도 바 미달을 정직하게 명시.** 재학습해도 미래 1주 R² 0.85~0.86. 오차는 레짐-온셋 주(147·113·109)에 집중 = 새 레짐은 보기 전엔 못 맞춘다.
4. **사전등록 규율의 자기검증 3회.** ① KFold로 골랐다면 정직축 최악(ET)을 뽑았을 것(M4) ② B2 시간분할이 사전 기록대로 −4.6의 미실현을 정확히 잡음(M6) ③ v14 조회 6회 전부 사전등록 내, 결과 후 게이트 수정 0 — 체리피킹 방지가 실제로 작동했다.

# Part II. 근거 사슬 — 결론이 어떻게 확정됐나

최종 결론은 단일 실험이 아니라 **7단계 수렴 증거**로 확정됐다. 각 단계는 독립적 방법(잔차 분해 / CV축 비교 / 시간분할 / 상관 안정성 진단 / ablation / 워크포워드 백테스트 / 정상성 변환 스크리닝)으로 같은 방향을 가리켰다.

```
[v13 M4.5]   잔차 78~98.7% Lot간 · 센서 무상관 0.037      "남은 신호는 센서로 접근 난망"
   ↓
[v13 M4/M5]  정직축(GKF C20) 맞대결 → XGB 챔피언           lot-CV에선 센서+튜닝이 −4.6 유효
   ↓
[v13 M6 B2]  시간분할: 챔피언 66.7→77~105 파국,            "같은 시기"용 이득이 미래엔 전이 안 됨
             70/30 컷에서 core10 단독에 열세                → G6′ [기각]
   ↓
[v13 R11]    원인: PM 유발 레짐 시프트(638→1217)            센서–타깃 corr 부호 반전 = 비정상성
             + 센서 관계 시간 비정상 / 백본은 안정           → 피처로 제거 불가, 재학습이 정공법
   ↓
[v13 R12]    시간불안정 14피처 제거(lean-85)해도             드리프트 프록시는 lot-CV 예측력 zero
             lot-CV 동급 → 파시모니만 확인
   ↓
[v13 B2′]    롤링-재학습 백테스트(홀드아웃 최종 회):          재학습해도 lean-85 < core10
             99.84 vs 98.36 · R² 0.854                      → 센서 운영 무가치 · v13 정직 종료
   ↓
[v14 V0~V2]  마지막 지렛대 = 센서의 "정상 성분"만 추출        P1~P4 전부 core10보다 악화(−0.5~−13.4)

```

**v14가 더한 결정적 조각**: P4(두단계: core10이 레벨 예측 → 센서는 잔차만 예측)는 센서에게 정상 성분만 요구한 설계인데 기여가 0에 못 미쳤다(−0.74). 즉 **레짐 레벨을 걷어낸 뒤 남는 성분에 센서 정보가 사실상 없다.** P1(−13.4)·P2(−8.1)의 큰 악화는 상대화가 드리프트만이 아니라 core10이 쓰던 주 신호(레벨)까지 지웠음을 보여준다.

# Part III. 운영 핸드오프 (정직 · 달성 가능 최선)

- **운영 권장 모델 = core10(10피처) + 주간(또는 PM 경계) 재학습.** 센서 불요. honest R² ≈ 0.86 · pooled RMSE ≈ 98. 재학습 필수 근거: 무재학습 254.9 vs 재학습 98.4.
- **한계 명시**: 레짐-온셋 주(PM 직후)는 어떤 모델도 예측 불가(147·113·109) — 재학습 후 회복. 이 구간은 **"예측 신뢰 낮음" 플래그**로 운영.
- **모니터링**: 입력 PSI · 잔차 lot-ICC · 주간 RMSE · PM 이벤트(pm_log) 트리거 재학습.
- **수치 인용 규칙**: lot-CV 축의 66~71은 "같은 시기 새 Lot"(후향 분석·대회축) 전용 — **미래를 엿본 낙관 수치이므로 현업 근거로 인용 금지.** 현업 인용 수치는 시간축 98.36 / R² 0.859.
- **팀 목표(70)에 대한 답**: 약 28pt 부족. 마지막 미검증 지렛대(P1~P4)를 다 써본 결과 격차가 줄기는커녕 벌어짐 → 현 데이터·현 센서로 도달 불가.
- **결론의 범위(정직 한계)**: 시간 견고성 근거는 학습기 내부 rolling-CV — 진짜 미래 확증은 새 미래 데이터 필요. 또한 본 결론은 **현재 웨이퍼-집계 FDC 피처 체계** 범위이며, 원시 트레이스 수준의 전혀 다른 표현(예: 물리 모델 기반 상태추정)까지 부정하지는 않는다.
- **잔여 옵션(비의사결정)**: valid/test same-era 1회 채점(R5)은 챔피언 부재로 판정 가치 없음 → 미실시 권장(기록용 부록만 가능, 드리프트 근거 인용 금지).

# 부록 C. 추가 검토 (2026-07-18) — floor 제약 완화로 천장 돌파 가능한가?

> 통합본 작성 후 추가된 질의 검토. 새 실험 없음 — 기존 원문 수치만으로 답한다.

**질의**: 필수 5센서(C17 온도·C11 플라즈마·C31 RF·C15/C16 가스) 각 ≥1 조건(floor, R10)을 해지하고, "센서 피처 단순 ≥1"로 완화하면 천장을 뚫을 여지가 있는가?

**답: 여지 거의 없음 — floor는 병목이 아니었다.**

1. **floor는 '포함 강제'지 '배제' 조건이 아니다.** 5센서 외 나머지는 GA가 이미 자유롭게 선택했고, 그 자유 선택분이 집어온 것이 C25 등 드리프트 프록시였다(REPORT_12: lot-CV 예측력 zero 판명). 즉 "제약 없는 선택이 무엇을 고르는지"는 이미 관측됐다.
2. **정량 상한이 너무 작다.** lot-CV에서 센서 89개 전체의 기여 = core10 대비 −0.47(67.451→66.978)이고, floor 5센서는 gain의 12.7%뿐. floor를 풀어 얻는 것은 이 envelope 안의 재배치인데, 천장(66.7~66.8)은 99·85·10피처 셋 무관하게 수렴했다 — 셋 구성의 문제가 아니다.
3. **시간축에선 방향이 반대다.** V2에서 센서는 어떤 형태(원값·P1~P4)로 넣어도 전부 core10보다 악화 → "센서 ≥1" 조건 자체가 손해 조건. 제약을 완전히 풀면 최적해는 **센서 0개 = core10(98.36)** — 이미 알고 있는 답으로 회귀한다.
4. **천장의 정체가 제약이 아니다.** 잔차의 78~98.7%가 Lot간 분산인데 센서와 무상관(0.037), 게다가 센서–타깃 관계가 시간 비정상(부호 반전). P4(레벨 걷어내고 잔차만 요구)마저 기여 0 미만 — 어떤 센서를 고르느냐의 문제가 아니라, 센서 신호에 남은 정보 자체가 없다.

**잔여 여지의 층위**: 피처 선택 규칙이 아니라 다른 층위에 있다 — 표현 수준 변경(원시 트레이스·물리 모델 기반 상태추정), 새 미래 데이터, 문제 재정의(레짐-온셋 감지로 전환 등). 또한 floor 해지는 §0 요건("센서가 실질 근거") 자체를 바꾸는 것이므로, 그 순간 v13/v14가 검증하던 질문과 다른 문제가 된다.
