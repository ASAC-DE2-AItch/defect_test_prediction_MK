## 0. 요건 & 성공 정의

### 0.1 배경 (v13 교훈 → v14 동기)
v13 종료 결론: **웨이퍼-집계 FDC 원시 피처는 미래-Lot 예측에 운영 가치 없음**(B2′에서 재학습해도 센서 lean-85 99.84 > 백본 core10 98.36, R² 0.85). 원인 = PM 레짐 시프트(C65 638→1217)로 **센서–타깃 관계가 시간에 비정상**(부호반전). → 남은 **유일한 미검증 지렛대 = "설계상 시간-정상성을 갖는 신규 피처"**. v14는 이 가설만 검증한다. **기대 성공확률은 낮음**(0.037 벽) — 시작부터 명시.

### 0.2 평가 축 (의사결정 언어)
| 축 | 정의 | 역할 |
|---|---|---|
| **① 시간축 (主)** | 학습기 내부 **rolling-origin time-CV**(확장창, 재학습, 단기 horizon 예측) | 모든 피처·모델·게이트의 **유일** 기준 |
| ①′ 벤치마크 | **core10 + 재학습**(v13 터미널 = REPORT_13, pooled 98.36·R² 0.85) | 넘어야 할 기준선 |
| ② 보조 | lot-CV(GKF C20) · valid/test **same-era 독립 채점**(R3, 1회) | 기록·독립 일반화 확인(드리프트 검증 아님) |

> **v13 대비 결정적 차이**: v13은 ②(lot-CV)로 골랐다 → v14는 ①(시간축)로만 고른다.

### 0.3 성공 정의 & 게이트 (사전등록)
| 요건 | 조작적 정의 |
|---|---|
| **드리프트 피처가 값을 하는가** | 신규피처+재학습 rolling time-CV pooled RMSE **< core10+재학습 − 3pt**(유의) |
| **정확도 바** | rolling time-CV pooled **R² ≥ 0.9**(v13이 0.85로 놓친 바) |
| **독립 일반화** | valid/test same-era 채점이 파국 아님(기록·부록) |
| **누수 0** | Lot/WF ID 비피처(R2), 인과적 시간분할, floor 유지 |

**최종 판정**: 위 2개(개선 ∧ R²≥0.9) 충족 → v14 성공(현업 후보). **미달 → v13 결론 확정·정직 종료**(센서로는 안 된다).

## 1. 핵심 전환 2가지

**전환 A — 축 교정 (시간축을 처음부터 의사결정 축으로).**
V0에서 rolling-origin time-CV 하니스를 세우고, **모든 피처 선택·모델 판정을 이 축으로**. lot-CV·valid는 기록만. (v13 T1 교훈)

**전환 B — 드리프트-겨냥 신규 피처 (재선별 아님).**
v13은 *원시 절대 센서값*을 집계했고 그게 달력·PM으로 드리프트했다. v14는 **설계상 정상성을 갖는 변환**을 새로 만든다(§3). 기존 셋 재선별(=lean-85)은 이미 실패했으므로 **반복 금지.**

## 2. 홀드아웃 규율 (정직한 제약)

- **fresh 미래 홀드아웃 없음**: R5(후기 조회 2회)는 v13 B2/B2′로 소진. **valid_X는 train과 같은 시기(2018-12~2019-02)** = same-era(미래 아님), test_X는 시각 파싱 불가. → **깨끗한 미래-시간 홀드아웃이 데이터셋에 없다.**
- **개발**: 학습기 내부 **nested rolling time-CV**로만 피처·모델 선택. 각 fold는 과거로 학습→단기 미래 예측(인과적).
- **독립 확인**: valid/test는 **same-era 독립 lot** 확인용(드리프트 검증 아님) — **1회 채점**(부록, 의사결정 불사용).
- **정직 한계 명시**: v14의 "시간 견고성"은 *학습기 내부 rolling-CV* 근거 — 진짜 미래 확증은 **새 미래 데이터가 있어야** 가능. 이 한계를 최종 보고에 못박는다.
- **이중딥핑 통제**: rolling-CV를 반복 조회하며 피처를 "맞출 때까지" 고르는 것 금지 — §7 중단 기준으로 시도 횟수 제한.

## 3. 신규 피처 패러다임 (설계상 정상성)

> 공통 원리: **절대값이 아니라 "레짐/PM/최근 기준 대비 상대값"** — 달력 드리프트를 구조적으로 제거.

| ID | 피처군 | 정의 | 왜 시간-정상적인가 |
|---|---|---|---|
| **P1** | PM-상대 | 센서 통계를 `days_since_last_pm` 구간(신품~노후)별로 정규화(z-score) | PM 사이클 위상 고정 → 노후 드리프트 상쇄 |
| **P2** | 레짐-조건부 | `is_high_regime`별 센서 baseline 차감(레짐 내 편차) | 레짐 레벨 제거 → 레짐 내 신호만 |
| **P3** | 정상성 변환 | 최근 롤링 baseline(직전 N Lot median) 대비 편차 = detrended 센서 | 달력 추세 제거(C25류 노후 드리프트 무력화) |
| **P4** | 타깃 분해 | 1단계 core10로 레짐 레벨 예측 → **2단계 센서로 잔차만** 예측 | 예측 대상을 정상 성분(잔차)으로 한정 |
| P5(조건부) | 신규 집계 | step 내 slope/delta/range 등 | M4.5상 기대 낮음 → P1~P4 결과 본 뒤만 |

- 각 피처군은 **독립 생성·독립 평가**(어느 원리가 실제로 시간-정상 신호를 담는지 분리 확인).
- floor(필수 5센서)·누수 규칙 v13 승계(R2·R10).

## 4. 로드맵

```
V0  시간축 하니스 + 벤치마크 재현
     └ rolling-origin time-CV 프레임(확장창·H=7d·재학습) 구현
     └ core10+재학습 벤치마크 재현(REPORT_13 pooled 98.36·R² 0.85 = self-check)
     └ valid/test same-era 로더(채점은 부록까지 보류)
V1  드리프트 피처 생성 (P1~P4, 웨이퍼 단위)
V2  시간축 피처 선택
     └ P1~P4를 rolling time-CV로 평가, core10+재학습 대비 개선 측정
     └ 피처군별 기여 분해(어느 원리가 값을 하나)
V3  챔피언 확정 + 게이트
     └ 최선 조합 rolling time-CV → §0.3 게이트(개선≥3 ∧ R²≥0.9)
V4  독립 확인 + 종결
     └ valid/test same-era 1회 채점(부록) → 성공 시 현업 후보·핸드오프 / 미달 시 정직 종료
```
예상: 순조 시 2~3일, 피처군 확장 포함 4~5일.

## 7. 중단 기준 (정직 종료 — 사전 선언)

- **2개 피처군 연속으로 core10 대비 무개선(<3pt)** → 중단.
- **최선 조합이 rolling time-CV R² < 0.9** → 중단(정확도 바 미달).
- 중단 시 결론 = **"v13 터미널(센서 운영 무가치) 확정, 운영 = core10+재학습"** 을 최종 보고에 못박고 M8 핸드오프로 복귀.
- **천장이 낮음을 알고 시작한다** — 무개선은 실패가 아니라 *데이터의 성질을 재확인한 정직한 데이터*다.
