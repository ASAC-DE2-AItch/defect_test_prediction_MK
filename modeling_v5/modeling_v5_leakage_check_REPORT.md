# modeling_v5 — Lot 누수 검증 결과 (멘토 피드백 대응)

> "성능이 비정상적으로 높다. valid로도 검증하라"에 대한 정직한 일반화 평가
>
> 실험일: 2026-07-07 (세션 4)

---

## 1. 배경 — 멘토 우려

v5의 RMSE ~60이 비정상적으로 높아 보임 → 누수 의심. valid 검증만으로는 부족한데, **valid/test가 train과 Lot을 99.9~100% 공유**하기 때문.

---

## 2. 문제 구조 — 간접 Lot 누수

Lot ID(C20)를 **피처로는 쓰지 않지만**, CV를 wafer(C64) 단위로 나눈 것이 문제.

| 근거 | 값 |
|------|-----|
| Lot당 wafer 수 | 평균 9.8장 |
| Lot 내부 C65 표준편차 | 26.5 (전체 262) → Lot만 알면 R²≈0.99 |
| GroupKFold(C64)에서 검증 wafer가 train에 같은 Lot 형제 보유 | **99.9%** |
| valid/test가 train과 Lot 공유 | 99.9% / 100% |

→ 모델이 ID 없이도 **센서 유사성으로 같은 Lot 패턴을 학습에서 엿봄**. valid도 독립이 아니므로 낙관적.

---

## 3. 검증 — wafer CV vs Lot CV

동일한 v5 피처로 CV 그룹키만 교체:

| CV 방식 | OOF RMSE | R² | 의미 |
|---------|----------|-----|------|
| **GroupKFold(C64=wafer)** | 62.63 | 0.943 | 현재 v5 방식. 같은 Lot 형제가 train에 존재 (낙관적) |
| **GroupKFold(C20=Lot)** | **70.42** | 0.928 | 통째로 새로운 Lot 예측 (신규 Lot 실제 성능) |
| **낙관 편향** | **+7.79 pt** | — | Lot 누수로 부풀려진 크기 |

---

## 4. 결론

1. **modeling_v5에 코드/구현 버그는 없다.** GroupKFold·타깃인코딩·ID 배제 모두 올바름.
2. **멘토 우려는 부분적으로 타당하다.** wafer 단위 평가가 신규 Lot 대비 ~8pt 낙관적.
3. **재앙적 누수는 아니다.** Lot ID 직접 사용 시 RMSE 30(R² 0.99)과 달리, 신규 Lot에서도 R² 0.93으로 여전히 유효.
4. **두 성능을 분리 보고해야 한다:**
   - 대회 리더보드 (valid/test, Lot 공유): RMSE ~60~62
   - 실무 신규 Lot (Lot 단위 CV): RMSE ~70

---

## 5. 권고 (앞으로의 평가 원칙)

- **최종 모델 CV는 GroupKFold(C20=Lot)로** — 실무형(신규 Lot) 일반화를 정직하게 평가. (Lot은 그룹키로만, 피처로는 계속 미사용.)
- 모든 버전(baseline~v7)의 성능표를 Lot 단위 CV로 재계산하면 "진짜 성능" 순위가 바뀔 수 있음 → 재평가 가치 있음.
- 목표 RMSE도 이 기준으로 재설정 필요: 신규 Lot 기준 ~70에서 개선을 논해야 함.
- 신뢰도(conformal/σ)도 Lot 단위 OOF로 재보정하면 신규 Lot에 더 정직한 구간이 됨.
