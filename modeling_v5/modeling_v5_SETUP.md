## 1. 이 프로젝트가 하려는 것

반도체(SK Hynix) 공정 중 웨이퍼(WF) 하나마다 센서 값(FDC Trace)이 여러 번(스텝마다) 기록됩니다.
이 센서 기록들을 보고, 그 웨이퍼가 최종적으로 받을 결함 검사 점수(C65)를 미리 예측하는 것이 목표입니다.

- 평가 방법: RMSE (예측값과 실제값의 오차 크기. 작을수록 좋음)
- 목표: RMSE ~40 (v5 기준 현재 약 61~62 -> 아직 미달성)

### 정리하면 최종적으로 쓴 피처

- **C6, C7**: 레시피/스텝 번호 -> LightGBM에 "범주형"이라고 알려줘서, 숫자 크기가 아니라 "종류"로 다루게 함

- **웨이퍼 전역 맥락(context)**: 같은 웨이퍼 안의 센서 평균/표준편차/최솟값/최댓값을 각 행에 함께 붙임 - 개별 스텝 정보 + 웨이퍼 전체 경향을 동시에 보여주기 위함
- **row_pos**: 웨이퍼 안에서 몇 번째 스텝인지
- **C33(PM time)**: 설비 정비 이후 경과 정도를 나타내는 지표로 활용

## 4. 모델 선택 이유

- **모델: LightGBM (트리 기반 회귀 모델)**
  - 이유: 표(테이블) 형태 데이터, 특히 센서처럼 컬럼이 많고 컬럼끼리 복잡하게 얽힌 데이터에서 가장 안정적으로 성능이 잘 나오는 모델 계열이기 때문. v1~v4에서도 계속 이 계열을 사용해왔고, 다른 대안보다 결과가 좋았음.
  - v5의 예측 단위 변경: 기존(v1~v4)은 "웨이퍼 1개 = 1행"으로 집계 후 예측했는데, 이 방식의 RMSE가 ~62에서 더 안 내려감(천장). v5는 "스텝 1개 = 1행"으로 예측한 뒤, 같은 웨이퍼에 속한 예측값들을 평균 내서 최종 웨이퍼 예측으로 씀. 위 3장에서 발견한 "스텝별로 센서 의미가 다르다"는 사실을 살리기 위함.
  - 검증 방법: **GroupKFold(C64 기준) 5-Fold** - 같은 웨이퍼의 행이 학습셋과 검증셋에 동시에 섞이면 정답을 미리 엿보는 셈이 되므로, 웨이퍼 단위로 통째로 나눠서 검증.
  - 하이퍼파라미터: 수동 설정(learning_rate=0.03, num_leaves=127 등). v3에서 썼던 Optuna 자동 튜닝은 v5에는 아직 적용 안 함(추가 개선 여지로 남겨둠).
  - v4에서 있었던 버그(`.values`로 변환해 컬럼명이 사라지는 문제) 방지를 위해 pandas DataFrame 형태 그대로 학습에 사용.

## 6. 모델 신뢰도(R^2 0.947, 상관계수 0.973)가 높게 나온 이유

v5 모델을 Test 데이터에 적용한 뒤 정답과 비교해보면, RMSE는 60.52로 목표(40)에 못 미치지만 R^2=0.947, 상관계수(corr)=0.973이라는 꽤 높은 수치가 나옵니다. "RMSE는 목표 미달인데 신뢰도는 왜 이렇게 높게 나오지?"라는 의문이 들 수 있어 근거를 정리합니다.

### 계산 근거

R^2(결정계수)는 "정답이 원래 가진 변동(분산) 중에서 모델이 설명해내는 비율"입니다. 공식은 다음과 같습니다.

```
R^2 = 1 - (모델 오차의 분산) / (정답 자체의 분산)
    ~= 1 - (RMSE / 정답의 표준편차)^2
```

실제 수치를 넣어보면:
- RMSE = 60.52
- 정답(C65)의 표준편차 ~= 262 (Test 기준)
- (60.52 / 262)^2 ~= 0.0533
- R^2 ~= 1 - 0.0533 ~= 0.947

즉 R^2가 높게 나온 것은 계산 실수나 과장이 아니라, **정답 값 자체의 변동폭(표준편차 262)이 모델의 오차(RMSE 60.52)보다 훨씬 크기 때문**입니다. 상관계수 0.973도 같은 맥락으로, "예측값이 정답의 오르내리는 큰 흐름(트렌드)은 매우 잘 따라간다"는 뜻입니다.

### 그런데 왜 이게 "목표 RMSE 40 미달"과 같이 있을 수 있나

- R^2/상관계수는 **상대적인 지표**(전체 변동 대비 얼마나 잘 맞췄는가)이고, RMSE는 **절대적인 지표**(오차가 실제로 몇 점인가)입니다. 두 지표가 다른 걸 재기 때문에, "큰 흐름은 거의 다 맞히지만(R^2 높음), 세부적인 절대 오차는 아직 40점대로 못 줄임(RMSE 미달)"이라는 상황이 동시에 성립할 수 있습니다.
- 실제로 목표 RMSE 40을 달성하려면 R^2가 약 0.977까지 올라가야 합니다((40/262)^2 ~= 0.0233, 1-0.0233~=0.977). 지금 0.947에서 0.977까지, 안 잡은 분산의 절반 이상을 추가로 잡아야 하는 셈입니다.
- 오차를 구간별로 뜯어보면 전반적으로 잘 맞히는 중에도 특정 구간(정답 742~976 구간)에서 편향(+58.8, 심한 과대예측)이 튀는 것이 확인됨 -> 이 구간이 R^2는 높지만 RMSE가 목표에 못 미치게 만드는 주요 원인 중 하나로 추정됩니다(모델이 이 구간의 어떤 특성을 놓치고 있음).

### 요약

| 지표 | 값 | 의미 |
|---|---|---|
| RMSE | 60.52 | 절대 오차 크기 (목표 40 대비 아직 부족) |
| R^2 | 0.947 | 정답 전체 변동 중 모델이 설명하는 비율 (매우 높음) |
| corr | 0.973 | 예측값과 정답의 전반적 추세 일치도 (매우 높음) |
| 목표 달성에 필요한 R^2 | ~0.977 | RMSE 40을 만족하려면 필요한 수준 |

결론: 모델은 전반적인 추세를 잘 잡고 있으나(R^2/corr 높음), 절대 오차를 더 줄이려면(RMSE 60->40) 지금 놓치고 있는 특정 구간(regime)의 신호를 추가로 잡아야 합니다.
