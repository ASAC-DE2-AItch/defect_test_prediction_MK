# modeling_v10 REPORT 01 — combined + Lot 타깃인코딩 (LotTE)

> 스냅샷 고정: 2026-07-11 (세션). 이후 분석이 바뀌면 새 REPORT를 만든다(이 파일은 수정 안 함).
> 모델: `modeling_v10.ipynb` — combined(192) + `C20_te` = 193피처, row(step)예측→WF평균, GroupKFold(C64) 5-fold LGBM.

---

## 1. 한 줄 요약

combined 피처(48.68)에 **Lot 단위 타깃인코딩 `C20_te`(그 Lot의 평균 C65) 딱 1개**를 추가해
**valid 34.29 / test 34.33** 달성 (−14pt). 현 챔피언(ExtraTrees 35.66/35.64)도 소폭 상회.
단 이 이득은 **valid/test가 train과 Lot을 공유해서 나오는 누수성 점수**다.

## 2. 성능

| 지표 | combined(192) 기준 | **v10 (+C20_te)** | Δ |
|------|------|------|------|
| OOF (GroupKFold C64) | 49.86 | **34.87** | −14.99 |
| valid | 48.68 | **34.29** | −14.39 |
| test | 48.65 | **34.33** | −14.32 |

OOF·valid·test가 **셋 다 34대로 일치** → 매핑이 안정적으로 전이됨(누수지만 '깨지지 않는' 종류).
대조: v9식 Lot 센서집계는 OOF만 좋고 valid 100+로 붕괴했음.

## 3. 오차 분석 (valid, n=1990)

- RMSE 34.29 · MAE 24.06 · **bias +0.59** (거의 무편향, 잘 보정됨)
- y(C65) 범위 511~1672, 평균 973.
- **5분위 RMSE (낮은 y → 높은 y): 21.7 / 28.7 / 26.6 / 34.0 / 52.3**
  → 불량이 **높은 웨이퍼일수록 오차 큼**. 최상위 구간(고불량)에서 RMSE가 평균의 1.5배.
- 최대오차 3건 모두 **양(+)의 오차 = 과소예측**: y=1666→pred 1111(+555), y=1343→1172(+171), y=1221→1056(+165).
- 해석: 극단 고불량 웨이퍼를 **Lot 평균 쪽으로 끌어내려 과소예측**(TE의 평균회귀 특성). 대량 불량 롯트 조기경보엔 이 tail 과소예측이 약점.

test도 동일 패턴(5분위 20.3→50.2, 최대오차 y=1562→1057(+505)).

## 4. 누수 해석 — 이 34점의 정체

- valid/test는 train과 **Lot을 99.9~100% 공유**(팀 `modeling_v5_leakage_check` 확인).
- `C20_te`는 train에서 만든 "Lot→평균C65" 지도를 valid/test에 조회하는 것 → 사실상 정답 레벨을 대부분 주입.
- **신규 Lot 기준(GroupKFold C20)으론 70~100대**로, 실무 일반화는 안 됨.
- 즉 v10은 **대회 리더보드 전용**. 프로젝트 원칙(`CLAUDE.md`: Lot ID 피처 금지)과 배치됨을 명시하고 채택.
- 근거 실험(같은 세션): v9 Lot 센서집계 OOF 34.8 / valid 142 (붕괴) ↔ Lot 타깃인코딩 OOF 34.9 / valid 34.3 (전이 성공). "Lot 정보를 어떻게 붙이냐"가 성패를 가름.

## 5. 결론

1. 대회 점수 목표(누수 감수) 하에서 v10은 현 최고 성능 라인(valid 34.29).
2. 약점: 고불량 tail 과소예측(평균회귀). 
3. 정직한 성능이 필요하면 combined(48.68)·챔피언(35.6) 라인 별도 유지.
