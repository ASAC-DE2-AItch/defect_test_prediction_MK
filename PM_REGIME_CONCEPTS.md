> **문서 성격**: 프로젝트 공용 용어·개념 정의서 (v8 이후 모든 문서·코드가 이 정의를 따른다)
> **작성일**: 2026-07-08 | **개정 v1.1** (2026-07-09): **조용(quiet)/요란(loud) 제2축 도입** (팀 크로스체크 회신). 레짐 트리거를 type(정비 내용)에서 **verdict(Qual 요란 판정)**로 교체, "minor→저불량 복귀" 가정 폐기("클리닝=현 레벨 유지" 팀 전제로 대체), §1.2·Q7 신설, Q4·Q6 재작성. **현 데이터에선 두 체계의 값이 동일**(유일 경계 12-24가 loud+major)하므로 레짐 통계·학습 값·성능 수치는 전부 불변
> **정의 출처**: 현업 정비 구분(2026-07-08 사용자 전달) + 팀 크로스체크 회신(2026-07-09, 조용/요란) + `EDA/02_EDA_regime_REPORT.md` 실측
> **한 줄 요약**: PM(정비)에는 **major(부품 교체)/minor(챔버 클리닝)** 라는 "정비 내용"이 있고, 각 이벤트에는 **요란(loud, 레짐을 바꿈)/조용(quiet, 안 바꿈)** 이라는 "결과 판정"(Qual이 D+0에 결정)이 붙는다. 웨이퍼 불량 수준의 상태 — **고불량/저불량 레짐** — 를 결정하는 것은 type이 아니라 **최근 loud 이벤트**다.

## 1. 용어 정의 — 3개 층위

한 PM 이벤트는 세 층위로 기술된다: **type**(무엇을 했나, 정비 기록) → **verdict**(레짐이 바뀌었나, Qual D+0 판정) → **상태**(그 결과 지금 어느 레벨인가). 피처가 소비하는 것은 **verdict뿐**이다.

### 1.1 PM type (정비 내용) — "무엇을 했나"

| 구분 | **major PM** | **minor PM** |
|------|-------------|--------------|
| **현업 정의** | **부품을 교체하는 정도의 큰 정비** (챔버 개방·대교란) | **단순 챔버 클리닝 정도의 작은 정비** |
| verdict 경향 (팀 전제) | 요란일 **수 있음** — 단 팀 전제상 **~70%는 quiet**(레짐 불변). type=major라고 점프를 예측하면 안 되는 이유 | 통상 **quiet** — 클리닝은 **현 레벨 유지** (부품이 만든 새 운전점을 클리닝이 되돌리지 못함) |
| C33(장비 카운터) | 리셋됨 | 리셋됨 |
| 데이터 내 관측 | **1회 — 2018-12-24** (12-23 08:32 마지막 웨이퍼 → 17.0시간 생산 공백 → 12-24 01:31 재개, C33 39→1). **verdict = loud** (레짐 전환 실측) | **1회 — 데이터 시작 직전(≈2018-11-29 추정)**. C33=3 시작으로 역산(+1.62/일). verdict는 관측 불가(quiet 추정) |
| 기록 위치 | `pm_log.json`: `[{"date": "2018-12-24", "type": "major", "verdict": "loud"}]` | 데이터 범위 밖이라 미기록 (시작 앵커 2018-12-01이 역할 대행) |

**핵심 두 가지.** ① 두 type 모두 C33을 리셋하므로 **C33 리셋만으로는 type도 verdict도 알 수 없다.** ② type은 정비 기록에서, **verdict는 PM 직후 Qual 판정에서** 온다 — 어느 쪽이든 C65(불량값)로 역추정·사후 수정하는 것은 금지다(타깃 누수).

### 1.2 verdict (조용/요란) — "레짐이 바뀌었나" (제2축, v1.1 신설)

| 구분 | **quiet (조용)** | **loud (요란)** |
|------|----------------|----------------|
| 의미 | 이 PM이 레짐(레벨)을 **바꾸지 않았다** — 현 상태 유지 | 이 PM이 레짐을 **전환시켰다** — 새 레벨로 점프 |
| 판정 주체·시점 | **Qual, PM 직후 D+0** (X 채널 신호 — C17 델타 등) | 동일 |
| 방향 | 해당 없음 (변화 없음) | **상승/하강 모두 가능** — loud는 "전환"이지 "악화"가 아님. 하강 loud의 상태 기록 규칙은 D7(팀 확인 대기) |
| 피처 효과 | `is_high_regime` **불변**, `high_regime_days` **연속** (앵커 유지) | `is_high_regime` 상태 갱신, `high_regime_days` 0부터 재시작 (새 loud 앵커) |
| type과의 관계 | quiet minor(통상 클리닝), **quiet major**(팀 전제 ~70%) 모두 존재 | 관측 1회(12-24)는 loud + major. loud minor도 논리적으로 배제 안 함 |

> ⚠ **팀 전제 딱지**: "major의 ~70%가 quiet", "클리닝=현 레벨 유지"는 **관측 n=0의 팀 세계관·시뮬레이터 전제**이지 데이터 사실이 아니다. 운영에서 첫 후속 PM이 관측되면 잔차 모니터링으로 판정·갱신한다.

### 1.3 레짐 (regime) — "결과 상태"

| 구분 | **저불량 레짐** | **고불량 레짐** |
|------|---------------|---------------|
| 의미 | 현재 레벨이 낮은 상태 | 현재 레벨이 높은 상태 — 12-24 loud 전환 이후 |
| 데이터 구간 | 2018-12-01 ~ 12-23 (WF 3,692개) | 2018-12-24 ~ 2019-02-08 (WF 8,247개) |
| C65 수준 | **평균 637.5 ± 40.1** (매우 안정, 완만한 하향 드리프트) | **평균 1123.5 ± 159.4** — 직후 ~1,398에서 시작해 주별 감쇠(1398→1202→1151→1099→1030→1006), 6주쯤 ~1,000 안정화 |
| C33 사이클 | 3 ~ 39 | 1 ~ 74 |
| 피처 값 | `is_high_regime = 0` | `is_high_regime = 1` |
| 다른 문서에서의 별칭 | "pre-PM", "저불량" | "post-PM", "급등" |

> ⚠️ **옛 표현 두 가지 주의**: ① "pre/post PM"(유무 기준)은 폐기 — 전 데이터가 어떤 PM의 "이후"라 무의미. ② v1.0의 "minor → 저불량 복귀"도 폐기 — 복귀는 minor(type)가 아니라 **하강 loud(verdict)** 로만 일어난다고 상정한다. `is_high_regime`이라는 이름은 유지하되, 의미상 **is_post_loud_pm**(마지막 loud 전환의 결과 상태)이다 — 관측 loud가 상승 1회뿐인 데이터 시대에는 두 해석이 동치.

## 2. 세 층위의 관계 — 한 장 그림

```
      minor PM (quiet 추정, ≈11-29)    major PM (loud!, 2018-12-24)        후속 PM (미래, 미관측)
 ────────┬─────────────────────────────┬──────────────────────────────────┬──────────────────────
          │      저불량 레짐             │        고불량 레짐                 │ quiet → 현 레벨 유지
  C65     │  ≈ 638 (std 40, 안정)       │  1398 → 주별 감쇠 → ≈1006 안정화   │        (클리닝·조용한 major)
  C33     │  3 → 39 (+1.62/일)          │  1 → 74 (+1.62/일)                │ loud  → 새 레벨 전환
  피처    │  is_high_regime = 0         │  is_high_regime = 1               │        (상승/하강은 Qual 판정, D7)
 ─────────┴─────────────────────────────┴──────────────────────────────────┴──────────────────────
                          ↑ 17시간 생산 공백 + C33 리셋 (탐지 근거) — Qual 판정: verdict="loud"
```

- 운영 패턴: 한 loud 주기 안에 quiet PM(클리닝, 조용한 major)이 여러 번 낄 수 있다 — 그동안 레벨은 유지되고 감쇠 곡선만 이어진다.
- **물리적 해석**: 요란 전환(관측 1회 = 부품 교체)은 챔버 대교란이라 직후 파티클·상태 불안정으로 불량이 급등했다가 안정화(seasoning)된다 → 감쇠 곡선. 클리닝은 챔버를 정돈하지만 **부품이 만든 새 운전점 자체는 되돌리지 못한다**(팀 전제) → 레벨 유지.

## 3. 모델 피처와의 연결 (v8 · PLAN v1.5 기준)

| 피처 | 정의 | 이 문서 개념과의 관계 |
|------|------|----------------------|
| `is_high_regime` | **상태기계**: 시작 0, PM의 `verdict=="quiet"`면 유지, `"loud"`면 갱신 | **레짐 스위치.** "어떤 정비였나(type)"가 아니라 "**레짐이 바뀌었나(verdict)**"를 따른다. quiet PM은 플래그를 건드리지 않음 |
| `high_regime_days` | `days_since_last_loud × is_high_regime` — **앵커 = 마지막 loud 이벤트** | 고불량 레짐 안 경과일(감쇠 곡선 x축). **앵커가 "마지막 임의 PM"이면 quiet PM에서 리셋돼 가짜 신품 스파이크(~1400) 버그** → loud 앵커로 연속성 보장 (PLAN v1.5 수정) |
| `days_since_last_pm` | 최근 PM(**종류·verdict 무관**)부터의 경과일 (float, 모든 PM에서 리셋) | "마지막 정비 후 시간" — 클리닝 주기 노화 신호로 유지 |
| `C33` | 장비 시간 카운터 (**모든 PM에서 리셋**, ≈ +1.62/일) | dslp 준중복. quiet PM에도 리셋되므로 "is_high=1 ∧ C33≈1 ∧ hrd 연속"이라는 미학습 조합을 만듦 — 잔여 오염 리스크(R12), 스모크 (a)로 실측 |
| ~~`is_post_pm`~~ / ~~`post_pm_days`~~ | (폐기) | "PM 유무" 기준 — 전 구간 1이 되어 오동작 |
| ~~`is_high_regime`(type 트리거)~~ | (v1.1에서 폐기) | "type==major" 기준 — 조용한 major에서 오지 않을 점프를 예측(케이스 ①), minor에서 잘못된 저불량 복귀를 예측(케이스 ③) |

**pm_log.json 운영 규칙 (P1, v1.1 갱신)**: 새 PM이 발생하면 `{"date", "type", "verdict"}` 엔트리 1개만 추가하면 모든 피처가 자동 대응된다. type은 **정비 기록에서**, verdict는 **PM 직후 D+0 Qual 판정에서** 입력하고, **생산 C65를 보고 거꾸로 정하거나 사후 수정하지 않는다** (타깃 누수 — "C65 역추정 금지"의 verdict 확장). loud-하강 대비 `state` 필드 예약은 D7 (파서는 state 부재 시 "loud→1" 폴백).

## 4. 자주 나오는 질문 (FAQ)

**Q1. PM은 장비를 좋게 만드는 건데 왜 major PM 후에 불량이 오르나?**
정비의 장기 효과와 단기 교란은 다르다. **관측된 major(12-24, loud)는** 챔버 상태를 크게 흔들어 직후 불량이 급등했고, 안정화(seasoning)되며 6주에 걸쳐 1398→1006으로 감쇠했다. 단, 팀 전제상 major의 ~70%는 **quiet**(레짐 불변)이므로 "major = 불량 급등"으로 일반화하면 안 된다 — 오르는 건 major가 아니라 **loud 전환**이다.

**Q2. major PM을 할 때마다 값이 계속(누적으로) 오르나?**
모델은 그렇게 가정하지 않는다. 첫째, quiet major(~70% 전제)는 애초에 상태를 바꾸지 않는다. 둘째, loud 전환이 와도 `is_high_regime`이 0/1 플래그(횟수 아님)라서 **모든 loud 주기를 같은 패턴(≈1400 시작 → 감쇠 → ≈1000)으로 교환 가능하게** 취급한다 — 누적 상승은 구조적으로 없다. 실제 다음 loud의 레벨이 이번과 다를 수 있는데(방향도 상승/하강 모두 가능), 그 레벨 차이는 모델이 아니라 **R2R 보정 + 재학습**의 몫이다 (원본 COLD 분석: 미경험 레짐 raw RMSE ~700대 → R2R 보정 ~90). 새 loud 직후엔 신뢰구간 확대 + 우선 실검사가 실무 권고다.

**Q3. 데이터에 PM이 몇 번 있나?**
2회다 — minor 1회(데이터 시작 직전, C33=3 시작으로 역산, verdict 관측 불가·quiet 추정) + major 1회(2018-12-24, **verdict=loud 실측**). X 컬럼(C33 리셋+생산 공백)으로 탐지되는 것은 후자뿐이며, valid/test에서 독립 탐지해도 같은 날짜가 나온다.

**Q4. 고불량→저불량 복귀는 검증됐나?**
**아니다(미관측).** 12-24 이후 C33 리셋이 0회라 데이터에 후속 PM 자체가 없다. 게다가 v1.1에서 복귀의 상정 경로가 바뀌었다: 구버전은 "다음 minor가 저불량으로 되돌린다"고 가정했지만, 팀 세계관은 **"클리닝은 현 레벨 유지 — 복귀는 하강 loud로만 가능"**이다. 즉 복귀는 quiet가 아니라 loud 이벤트이고, 미관측 loud의 레벨 예측은 원래부터 모델이 아닌 R2R·재학습 영역이다(Q2). `is_high_regime`의 quiet/loud 거동 자체는 스모크 테스트 시나리오 3종(PLAN A-4)과 증강 시뮬레이션(M8, 팀 시뮬레이터 전제 준수)으로 검증한다.

**Q5. C33 리셋이 보이면 그게 어떤 PM인지 어떻게 아나?**
데이터만으로는 모른다. C33은 type·verdict 무관 모든 PM에서 리셋된다. **type은 정비 기록, verdict는 D+0 Qual 판정이 정본**이며, 12-24를 loud로 확정한 근거도 당시 Qual이 봤을 X 채널 신호(C17 델타 등)와의 정합이다 — C65 급등은 사후 정합 확인용이었고, 앞으로도 C65로 verdict를 역추정·사후 수정하지 않는다.

**Q6. 새로운 minor PM(클리닝)이 오면, 그 웨이퍼들의 예측값은 어느 범위에서 나오나?**
v1.1 세계관에선 **"지금 있는 레벨 그대로"가 정답 거동이다** — minor는 통상 quiet라 레짐을 바꾸지 않는다. 구버전 답("저불량 밴드로 복귀")은 폐기됐다. 구체적으로: ① **저불량 중** 클리닝 → 저불량 밴드(≈638) 유지. ② **고불량 중** 클리닝 → **현 감쇠 레벨 유지**(예: D+50이면 ≈1005) — 638 복귀도, ~1400 재스파이크도 오작동이다. 트리 출력은 여전히 학습 범위(503~1678) 안이며 `is_high_regime`(불변)이 라우팅을 지배한다. 단서: C33은 quiet PM에도 리셋되는 장비 카운터라 "is_high=1 ∧ C33≈1 ∧ hrd 연속"은 미학습 조합 — hrd를 못 본 일부 트리(feature_fraction)가 C33 프록시로 초기-major로 오인해 위로 끌 이론적 위험이 있고(R12), **스모크 테스트 (a)/(b) 예측 밴드 체크**로 실측 검증한다. 그리고 모델이 레벨을 유지하는 것과 **실제** 클리닝 후 Y가 유지되는 것은 별개다(팀 전제, 미관측) — 어긋나면 잔차 모니터링 → R2R 오프셋 흡수 → 재학습으로 대응한다(Q2와 동일 구조).

**Q7. (v1.1 신설) 조용/요란과 고불량/저불량은 뭐가 다른가? major/minor와는?**
축이 다르다. **major/minor = 정비 내용**(무엇을 했나 — 부품 교체 vs 클리닝, 정비 기록), **quiet/loud = 이벤트 판정**(그 정비가 레짐을 바꿨나 — Qual이 D+0 결정), **고불량/저불량 = 상태**(그 결과 지금 어느 레벨인가). 관측된 유일한 전환(12-24)이 "major이면서 loud이면서 상승"이라 세 체계가 지금은 완전히 포개져 있고, 그래서 학습 데이터 값은 어느 기준으로 계산해도 같다. 갈라지는 미래 케이스 3종: ① **조용한 major** (부품을 갈았지만 레짐 불변, 팀 전제 ~70%) — type 기준이면 오지 않을 점프를 예측, verdict 기준이면 정상. ② **요란한 하강** (전환이 개선 방향) — "고불량"이라는 상태 이름이 안 맞는 경우, 상태 기록 규칙은 D7. ③ **major 후 minor** — 구버전은 "저불량 복귀", 신세계관은 "현 레벨 유지" (스모크 (a)가 이 케이스).
